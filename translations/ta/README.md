# மொழிபெயர்ப்பாளர்-ஏஜென்ட்

₹8,30,000 மொழிபெயர்ப்பு சிக்கல், 90 வினாடிகளில் தீர்க்கப்பட்டது.

நிறுவனங்கள் தங்கள் தளங்களை உள்ளூர்மயமாக்க ஏஜென்சிகளுக்கு வார்த்தைக்கு ₹8–21 செலுத்துகின்றன. 5,000 வார்த்தைகள் கொண்ட தளத்தை 10 மொழிகளில் மாற்ற ₹4,15,000–9,95,000 செலவாகும் மற்றும் 2–4 வாரங்கள் ஆகும். எப்போதெல்லாம் நீங்கள் ஒரு தலைப்பை மாற்றினாலும், கணக்கு மீண்டும் ஆரம்பிக்கிறது.

இந்த கருவி அதை ஒரே கட்டளையில், 71 மொழிகளில், உங்கள் பில்ட் நேரத்திலேயே செய்கிறது:

```bash
bunx translator-agent -s ./dist -l all
```

ஏஜென்சி இல்லை. ஸ்ப்ரெட்ஷீட்கள் இல்லை. விற்பனையாளர் பிணைப்பு இல்லை. பதிவு இல்லை. உங்கள் விசைகள், உங்கள் பில்ட், உங்கள் மொழிகள்.

> **நீங்கள் இப்போது ஆதாரத்தைப் படித்துக் கொண்டிருக்கிறீர்கள்.** இந்த README `bunx translator-agent -s README.md -l all` இயக்கி மொழிபெயர்க்கப்பட்டது. போய் [ஜப்பானிய பதிப்பைப்](./translations/ja/README.md) படியுங்கள் — அது வெறுமனே "கணக்கு மீண்டும் ஆரம்பிக்கிறது" என்பதை மொழிபெயர்க்கவில்லை, அதற்கு பதிலாக ஜப்பானிய வணிக மொழிச்சொலிவைப் பயன்படுத்தியது. [ஜெர்மன் பதிப்பு](./translations/de/README.md) 30% நீளமாக உள்ளது, ஏனென்றால் ஜெர்மன் எப்போதும் அப்படித்தான். [அரபிக் பதிப்பு](./translations/ar/README.md) வலமிருந்து இடமாகப் படிக்கப்படுகிறது. ## இது ஏன் வேலை செய்கிறது

மொழிபெயர்ப்பு தீர்க்கப்பட்ட பிரச்சனை. உள்ளூர்மயமாக்கல் இல்லை.

கூகிள் மொழிபெயர்ப்பு "எங்கள் வெள்ளெலிகள் அதைச் செய்ய வேலை பார்த்துக் கொண்டிருக்கின்றன" என்பதை ஜப்பானியமாக மாற்றும். அது செய்ய முடியாதது என்னவென்றால், இந்த நகைச்சுவை ஜப்பானில் வெற்றிபெறாது என்பதை அடையாளம் கண்டு, அதற்கு பதிலாக உண்மையில் பொருத்தமான ஒன்றுடன் மாற்றுவது — இன்ஜினீயரிங் குழு இரவு முழுவதும் வேலை செய்வதைப் பற்றிப் பேசுவது போல, இது கலாச்சாரப்படியாகவும் பொருத்தமானது மற்றும் பின்னணியில் வேடிக்கையானதுமாக உள்ளது.

இந்த கருவி மொழிபெயர்ப்பு செய்யாது. இது **மறுபரிமாணம் செய்கிறது** — அதே செயல்முறை, விளம்பர ஏஜென்சிகள் ஒரு பிரச்சாரத்தை சந்தைகளில் மாற்றியமைக்க ₹41,50,000 வசூலிக்கின்றன. ஆனால் LLM ஏற்கனவே ஒவ்வொரு கலாச்சாரம், ஒவ்வொரு மொழிச்சொல், ஒவ்வொரு வடிவமைப்பு மரபுகளையும் அறியும். இது இதையெல்லாம் அறியும்:

- `$49/month` ஜப்பானில் `月額6,980円` ஆகிறது — yen குறியீட்டை ஒட்டிய "$49" அல்ல
- கிண்டல் ஆங்கிலத்தில் வெற்றி மற்றும் முறையான ஜப்பானியத்தில் தோல்வி
- "Drowning in paperwork" பிரெஞ்சில் "noyade administrative" ஆகிறது — ஒரு உண்மையான பிரெஞ்சு வெளிப்பாடு, வார்த்தைக்கு வார்த்தை மொழிபெயர்ப்பு அல்ல
- ஜெர்மானியர்கள் வெள்ளெலி நகைச்சுவையை வைத்திருக்கிறார்கள், ஏனென்றால் Hamsterrad (hamster wheel) ஒரு உண்மையான ஜெர்மன் மொழிச்சொல்
- பிரேசிலியர்களுக்கு சாதாரண பதிவு தேவை, இல்லையென்றால் ரோபோ எழுதியது போல் ஒலிக்கிறது

மாதிரி ஒவ்வொரு சரத்தையும் வகைப்படுத்துகிறது. UI லேபிள்கள் நேரடி மொழிபெயர்ப்பு பெறுகின்றன. சந்தைப்படுத்தல் நகல் மாற்றியமைக்கப்படுகிறது. நகைச்சுவை இலக்கு கலாச்சாரத்திற்காக முழுமையாக மறுபரிமாணம் செய்யப்படுகிறது.

## நீங்கள் அதை இயக்கும்போது என்ன நடக்கிறது

உங்கள் பில்ட் அவுட்புட்டில் அதைச் சுட்டிக் காட்டுங்கள். இது ஒவ்வொரு மொழியிலும் முழு கோப்பு மரத்தையும் நகலெடுக்கிறது — உரை கோப்புகளை மொழிபெயர்த்து, நிலையான சொத்துக்களை நகலெடுத்து, வரிசைப்படுத்தலுக்குத் தேவையான அனைத்தையும் உருவாக்குகிறது:

```
your-site/                          translations/
  index.html                          middleware.ts        ← மொழி அடையாளம்
  about.html             →            _locales.css         ← ஸ்கிரிப்டுக்கான அச்சுக்கலை
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", மறுபரிமாணம்
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 வரி உயரம்
                                        css/style.css      ← நகல்
                                        js/app.js          ← நகல்
                                        images/logo.png    ← நகல்
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% எழுத்துரு
                                        ...
                                      de/
                                        ...
```

ஒவ்வொரு HTML கோப்பும் `lang` மற்றும் `dir="rtl"` உட்செலுத்தப்படுகிறது. ஒவ்வொரு மொழியும் சரியான எழுத்துரு அடுக்கு, வரி உயரம் மற்றும் உரை திசையுடன் CSS பெறுகிறது. `Accept-Language` ஐப் படித்து சரியான மொழிக்கு மீண்டும் எழுதும் Vercel middleware உருவாக்கப்படுகிறது.

Vercel இல் வரிசைப்படுத்துங்கள். டோக்கியோவில் உள்ள பயனர் 1.8 வரி உயரத்தில் Hiragino Sans உடன் ஜப்பானிய மொழியைப் பார்க்கிறார். கெய்ரோவில் உள்ள பயனர் 110% அளவில் Noto Naskh உடன் RTL அரபிக் மொழியைப் பார்க்கிறார். பாங்காக்கில் உள்ள பயனர் `word-break: keep-all` உடன் தாய்மொழியைப் பார்க்கிறார், ஏனென்றால் தாய்மொழியில் இடைவெளிகள் இல்லை. கட்டமைப்பு இல்லை.

## உங்கள் சொந்த விசைகளைக் கொண்டுவாருங்கள் அல்லது வேண்டாம்

```bash
# உங்களிடம் விசைகள் உள்ளன — உள்ளூரில் இயங்குகிறது, நீங்கள் உங்கள் LLM வழங்குநருக்கு நேரடியாக பணம் செலுத்துகிறீர்கள்
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# உங்களிடம் விசைகள் இல்லை — வெறுமனே வேலை செய்கிறது
# தானாக hosted சேவையைப் பயன்படுத்துகிறது
# x402 மூலம் USDC உடன் மொழிபெயர்ப்புக்கு பணம் செலுத்துங்கள் — பதிவு இல்லை, கணக்கு இல்லை
bunx translator-agent -s ./dist -l all
```

அதே கட்டளை. API விசைகள் இருந்தால், அது உங்கள் வழங்குநருடன் உள்ளூரில் இயங்குகிறது. இல்லையென்றால், அது hosted API ஐத் தாக்கி [x402](https://x402.org) மூலம் கோரிக்கைக்கு பணம் செலுத்துகிறது — HTTP 402 பேமென்ட் புரோட்டோகால். உங்கள் கிளையன்ட் Base இல் USDC செலுத்துகிறது, மொழிபெயர்ப்புகளைப் பெறுகிறது. அங்கீகாரம் இல்லை, விற்பனையாளர் உறவு இல்லை, விலைப்பட்டியல் இல்லை.

Anthropic மற்றும் OpenAI ஐ ஆதரிக்கிறது. நீங்கள் விரும்பும் எந்த மாதிரியையும் கொண்டுவாருங்கள்:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## கேசிங்

மொழிபெயர்ப்புகள் பில்ட் கலைப்பொருட்கள். பில்ட் நேரத்தில் உருவாக்குங்கள், அவுட்புட்டைக் கேச் செய்யுங்கள், மூலம் மாறாதபோது தவிர்க்கவும்.

### Vercel

Vercel தானாக பில்ட் அவுட்புட்டைக் கேச் செய்கிறது. `postbuild` சேர்த்து முடித்துவிடுங்கள்:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

மூலம் மாற்றமில்லாமல் = கேச் ஹிட் = பூஜ்யம் LLM அழைப்புகள் = பூஜ்யம் செலவு.

## விருப்பங்கள்

```
பயன்பாடு: translator-agent [options]

விருப்பங்கள்:
  -s, --source <path>      ஸ்கேன் செய்ய மூல டைரக்ட்டரி அல்லது கோப்பு
  -l, --locales <locales>  இலக்கு மொழிகள், கமாவால் பிரிக்கப்பட்ட அல்லது 71 மொழிகளுக்கு "all"
  -o, --output <path>      அவுட்புட் டைரக்ட்டரி (முன்னிருப்பு: "./translations")
  -p, --provider <name>    anthropic | openai (முன்னிருப்பு: "anthropic")
  -m, --model <id>         மாதிரி மேலெழுதல்
  -c, --concurrency <n>    அதிகபட்ச இணையான LLM அழைப்புகள் (முன்னிருப்பு: 10)
  --api-url <url>          hosted சேவை URL (API விசைகள் அமைக்கப்படாதபோது தானாக பயன்படுத்தப்படும்)
```

| நீட்சி | உத்தி |
|---|---|
| `.json` | மதிப்புகளை மொழிபெயர்க்கவும், விசைகளைப் பாதுகாக்கவும் |
| `.md` / `.mdx` | உரையை மொழிபெயர்க்கவும், தொடரியல் பாதுகாக்கவும் |
| `.html` / `.htm` | உரையை மொழிபெயர்க்கவும், டேக்குகளைப் பாதுகாக்கவும், `lang`/`dir` உட்செலுத்தவும் |
| மற்ற அனைத்தும் | ஒவ்வொரு மொழி டைரக்ட்டரியிலும் நகலெடுக்கவும் |

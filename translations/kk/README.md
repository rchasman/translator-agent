# translator-agent

10 000 доллар локализация мәселесі 90 секундта шешілді.

Компаниялар сайттарын локализациялау үшін агенттіктерге сөз басына $0.10–0.25 төлейді. 5000 сөзді 10 тілге аудару $5000–12000 тұрады және 2–4 апта уақыт алады. Тақырыпты өзгерткен сайын есептеуіш қайта басталады.

Бұл құрал мұны бір командамен, 71 тілге, сіздің құрылыс қадамында жасайды:

```bash
bunx translator-agent -s ./dist -l all
```

Агенттік жоқ. Электрондық кестелер жоқ. Жеткізушіге тәуелділік жоқ. Тіркелу жоқ. Сіздің кілттеріңіз, сіздің құрылысыңыз, сіздің тілдеріңіз.

## Неліктен бұл жұмыс істейді

Аударма шешілген мәселе. Локализация емес.

Google Translate "Біздің хомяктар осы мәселелерді шешуде" деген сөзді жапон тіліне аудара алады. Ол істей алмайтын нәрсе — бұл әзілдің Жапонияда ұнамайтынын тану және оны жұмыс істейтін нәрсемен ауыстыру — мысалы, инженерлік топ түні бойы жұмыс істеуін сипаттау, бұл мәдени тұрғыдан лайықты және мәнмәтінде күлкілі.

Бұл құрал аудармайды. Ол **қайта жасайды** — осы процесс үшін жарнама агенттіктері науқанды нарықтар арасында бейімдеген кезде $50 000 алады. Дегенмен LLM әрбір мәдениетті, әрбір идиоманы, әрбір форматтау дәстүрін біледі. Ол мыналарды біледі:

- `$49/ай` Жапонияда `月額6,980円` болады — иен белгісі жапсырылған "$49" емес
- Сарказм ағылшын тілінде өледі және ресми жапон тілінде жойылады
- "Қағаз жұмыстарында батып бару" француз тілінде "noyade administrative" болады — нақты француз тіркесі, сөзбе-сөз аударма емес
- Неміс тілінде хомяк әзілі сақталады, өйткені Hamsterrad (хомяк дөңгелегі) нақты неміс идиомасы
- Бразилиялықтарға кәдімгі стиль керек, әйтпесе робот жазғандай естіледі

Модель әрбір жолды жіктейді. UI белгілері тікелей аударма алады. Маркетингтік мәтін бейімделеді. Юмор мақсатты мәдениет үшін толығымен қайта жасалады.

## Сіз оны іске қосқан кезде не болады

Оны құрылыс нәтижесіне бағыттаңыз. Ол локаль бойынша файл ағашының толық көшірмесін жасайды — мәтін файлдарын аударады, статикалық активтерді көшіреді және орналастыру үшін қажет барлық нәрсені генерациялайды:

```
your-site/                          translations/
  index.html                          middleware.ts        ← локаль анықтау
  about.html             →            _locales.css         ← жазу бойынша типография
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", қайта жасалған
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 жол биіктігі
                                        css/style.css      ← көшірілген
                                        js/app.js          ← көшірілген
                                        images/logo.png    ← көшірілген
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% қаріп
                                        ...
                                      de/
                                        ...
```

Әрбір HTML файлға `lang` және `dir="rtl"` енгізіледі. Әрбір локаль дұрыс қаріп стегімен, жол биіктігімен және мәтін бағытымен CSS алады. `Accept-Language` оқитын және дұрыс локальға қайта жазатын Vercel middleware генерацияланады.

Vercel-ге орналастырыңыз. Токиодағы пайдаланушы 1.8 жол биіктігімен Hiragino Sans қарпімен жапон тілін көреді. Каирдағы пайдаланушы 110% өлшемде Noto Naskh қарпімен RTL араб тілін көреді. Бангкоктағы пайдаланушы `word-break: keep-all` қасиеті бар тай тілін көреді, өйткені тай тілінде бос орын жоқ. Конфигурация жоқ.

## 90 секунд, 4 апта емес

```bash
# Үш тіл, бір JSON файл
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
Дайын. 3 файл 9.5 секундта жазылды

# Сіздің бүкіл сайт, жердегі әрбір тіл
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
Дайын. 142 файл аударылды, 284 статикалық файл 94 секундта көшірілді
```

## Өз кілттеріңізді алып келіңіз немесе келтірмеңіз

```bash
# Сізде кілттер бар — жергілікті жұмыс істейді, LLM провайдеріңізге тікелей төлейсіз
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# Сізде кілттер жоқ — тек жұмыс істейді
# Орналастырылған қызметті автоматты түрде пайдаланады
# x402 арқылы USDC көмегімен аударма үшін төлеңіз — тіркелусіз, аккаунтсыз
bunx translator-agent -s ./dist -l all
```

Бірдей команда. API кілттері болса, провайдеріңізбен жергілікті жұмыс істейді. Болмаса, орналастырылған API-ға соғады және [x402](https://x402.org) арқылы сұрау бойынша төлейді — HTTP 402 төлем хаттамасы. Сіздің клиентіңіз Base-да USDC төлейді, аудармаларды қайтарып алады. Авторизация жоқ, жеткізуші қатынас жоқ, шот-фактуралар жоқ.

Anthropic және OpenAI қолдайды. Қалаған үлгіңізді алып келіңіз:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Әрбір жазу жүйесі өңделген

Құрал тек мәтінді аудармайды — әрбір жазу жүйесінің қалай көрсетілетінін біледі:

| Жазу | Не өзгереді | Неліктен |
|---|---|---|
| **Араб, иврит, парсы, урду** | `dir="rtl"`, RTL қаріптер, 110% өлшем | Араб тілінде оқылатын үлкен қаріп керек; бүкіл макет айналады |
| **Жапон, қытай, корей** | CJK қаріп стектері, 1.8 жол биіктігі | Таңбалар тұрақты ені шаршылар; тік тыныс орны керек |
| **Хинди, бенгали, тамил, телугу** | Үнді қаріптері, 1.8 жол биіктігі | Басы сызықтар (широрекха) қосымша тік орын керек |
| **Тай** | `word-break: keep-all` | Сөздер арасында бос орын жоқ — браузерге нақты жол үзу ережелері керек |
| **Бирма** | 2.2 жол биіктігі | Кез келген негізгі жазудың ең биік глифтері |
| **Кхмер** | 2.0 жол биіктігі | Жазба дауыссыз кластерлер тігінен жиналады |

Локаль бойынша генерацияланған CSS:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Кэштеу

Аудармалар құрылыс артефактілері. Құрылыс уақытында генерациялаңыз, шығуды кэштеңіз, көз өзгермеген кезде өткізіп жіберіңіз.

### Vercel

Vercel құрылыс шығуын автоматты түрде кэштейді. `postbuild` қосыңыз, дайын:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Көз өзгертілмеген = кэш тиген = нөлдік LLM қоңыраулары = нөлдік құн.

## Опциялар

```
Қолдану: translator-agent [опциялар]

Опциялар:
  -s, --source <жол>       сканерлеу үшін көз каталогы немесе файл
  -l, --locales <локальдар> мақсатты локальдар, үтірмен бөлінген немесе 71 тіл үшін "all"
  -o, --output <жол>       шығыс каталогы (әдепкі: "./translations")
  -p, --provider <ат>      anthropic | openai (әдепкі: "anthropic")
  -m, --model <id>         модель ауыстыру
  -c, --concurrency <n>    максимум параллель LLM қоңыраулары (әдепкі: 10)
  --api-url <url>          орналастырылған қызмет URL-і (API кілттері орнатылмаған кезде автоматты пайдаланылады)
```

| Кеңейтім | Стратегия |
|---|---|
| `.json` | Мәндерді аудару, кілттерді сақтау |
| `.md` / `.mdx` | Мәтінді аудару, синтаксисті сақтау |
| `.html` / `.htm` | Мәтінді аудару, тегтерді сақтау, `lang`/`dir` енгізу |
| Барлық басқа | Әрбір локаль каталогына көшіру |

### Барлық 71 локаль

`-l all` интернет пайдаланушыларының ~95%-ын қамтиды: zh-CN, zh-TW, ja, ko, vi, th, id, ms, fil, my, hi, bn, ta, te, mr, gu, kn, ml, pa, ur, fa, tr, he, ar, kk, uz, fr, de, es, pt, pt-BR, it, nl, ca, gl, sv, da, no, fi, is, pl, cs, sk, hu, ro, bg, hr, sr, sl, uk, ru, lt, lv, et, el, ga, sw, am, ha, yo, zu, af, km, lo, ne, si, ka, az, mn

# translator-agent

₹7.5 लाखाची भाषांतर समस्या, ९० सेकंदात सोडवली.

कंपन्या त्यांच्या साइट्सच्या स्थानिकीकरणासाठी एजन्सींना शब्दामागे ₹8–20 देतात. ५,००० शब्दांची साइट १० भाषांमध्ये करण्यासाठी ₹40,000–1,00,000 लागतात आणि २–४ आठवडे वेळ. तुम्ही जेव्हाही शीर्षक बदलता, तेव्हाही पैसे चुकवावे लागतात.

हे टूल एका कमांडमध्ये, ७१ भाषांमध्ये, तुमच्या बिल्ड स्टेपमध्येच करून देते:

```bash
bunx translator-agent -s ./dist -l all
```

एजन्सी नको. एक्सेल फाइल नको. कुठल्याही कंपनीवर अवलंबित्व नको. साइनअप नको. तुमच्या की, तुमचा बिल्ड, तुमच्या भाषा.

> **तुम्ही त्याचा पुरावा वाचत आहात.** हा README `bunx translator-agent -s README.md -l all` चालवून भाषांतरित केला आहे. जा [जपानी आवृत्ती](./translations/ja/README.md) वाचा — त्यात फक्त "the meter restarts" चे भाषांतर नाही, तर ते जपानी व्यावसायिक म्हणीने बदलले आहे. [जर्मन आवृत्ती](./translations/de/README.md) ३०% लांब आहे कारण जर्मन नेहमीच तसे असते. [अरबी आवृत्ती](./translations/ar/README.md) उजव्याहून डाव्याकडे वाचली जाते. ## हे का कार्य करते

भाषांतर सोडवलेली समस्या आहे. स्थानिकीकरण नाही.

गूगल ट्रान्स्लेट "Our hamsters are working on it" चे जपानी भाषांतर करू शकते. परंतु ते असे ओळखू शकत नाही की हा विनोद जपानमध्ये काम करत नाही, आणि त्याच्या जागी असे काहीतरी घालू शकत नाही जे तिथे काम करेल — जसे की इंजिनिअरिंग टीम रात्रभर काम करत असल्याचा उल्लेख, जे सांस्कृतिकदृष्ट्या योग्य आणि संदर्भात मजेशीर आहे.

हे टूल भाषांतर करत नाही. ते **ट्रान्सक्रिएट** करते — तीच प्रक्रिया ज्यासाठी जाहिरात एजन्सी मार्केट्समध्ये मोहीम स्वीकारताना ₹40 लाख आकारतात. परंतु LLM ला आधीच सर्व संस्कृती, सर्व म्हणी, सर्व फॉर्मॅटिंग पद्धती माहीत आहेत. त्याला माहीत आहे की:

- `$49/month` जपानमध्ये `月額6,980円` होते — येन चिन्हासह "$49" नाही
- व्यंग इंग्रजीमध्ये काम करते आणि औपचारिक जपानीमध्ये मरते
- "Drowning in paperwork" फ्रेंचमध्ये "noyade administrative" होते — खरी फ्रेंच अभिव्यक्ती, शब्दाशी भाषांतर नाही
- जर्मन हॅमस्टर विनोद ठेवतात कारण Hamsterrad (हॅमस्टर व्हील) ही खरी जर्मन म्हणी आहे
- ब्राझिलियन्सला अनौपचारिक रजिस्टर हवे नाहीतर असे वाटते रोबोटने लिहिले आहे

मॉडेल प्रत्येक स्ट्रिंगचे वर्गीकरण करते. UI लेबल्सचे थेट भाषांतर होते. मार्केटिंग कॉपी स्वीकारली जाते. विनोद लक्ष्य संस्कृतीसाठी पूर्णपणे पुन्हा तयार केला जातो.

## जेव्हा तुम्ही ते चालवता तेव्हा काय होते

तुमच्या बिल्ड आउटपुटकडे निर्देश करा. ते प्रत्येक लोकेलसाठी संपूर्ण फाइल ट्री क्लोन करते — टेक्स्ट फाइल्सचे भाषांतर करते, स्टॅटिक अॅसेट कॉपी करते, आणि तैनातीसाठी आवश्यक सर्वकाही तयार करते:

```
your-site/                          translations/
  index.html                          middleware.ts        ← लोकेल शोध
  about.html             →            _locales.css         ← स्क्रिप्ट प्रमाणे टायपोग्राफी
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ट्रान्सक्रिएटेड
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← कॉपी केले
                                        js/app.js          ← कॉपी केले
                                        images/logo.png    ← कॉपी केले
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% फॉन्ट
                                        ...
                                      de/
                                        ...
```

प्रत्येक HTML फाइलमध्ये `lang` आणि `dir="rtl"` इंजेक्ट केले जाते. प्रत्येक लोकेलला योग्य फॉन्ट स्टॅक, लाइन-हाईट आणि टेक्स्ट दिशेसह CSS मिळते. Vercel मिडलवेअर तयार केले जाते जे `Accept-Language` वाचते आणि योग्य लोकेलमध्ये रिराइट करते.

Vercel वर तैनात करा. टोकियोमधील वापरकर्त्याला १.८ लाइन-हाईट असलेले Hiragino Sans सह जपानी दिसते. कैरोमधील वापरकर्त्याला ११०% साईज असलेले Noto Naskh सह RTL अरबी दिसते. बँकॉकमधील वापरकर्त्याला `word-break: keep-all` असलेले थाई दिसते कारण थाईमध्ये स्पेस नसतात. कॉन्फिग नको.

## ९० सेकंद, ४ आठवडे नाही

```bash
# तीन भाषा, एक JSON फाइल
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
पूर्ण झाले. 3 फाइल्स 9.5 सेकंदात लिहिल्या

# तुमची संपूर्ण साइट, पृथ्वीवरील प्रत्येक भाषा
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
पूर्ण झाले. 142 फाइल्सचे भाषांतर, 284 स्टॅटिक फाइल्स 94 सेकंदात कॉपी केल्या
```

### तुमच्या बिल्ड पाइपलाइनमध्ये

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

प्रत्येक तैनाती ७१ भाषांमध्ये पाठवली जाते. भाषांतरे बिल्ड artifacts आहेत — कॅश केली, फक्त स्रोत बदलल्यावरच पुन्हा तयार केली.

## स्वत:च्या की घ्या किंवा घेऊ नका

```bash
# तुमच्याकडे की आहेत — स्थानिक चालते, तुम्ही थेट LLM प्रदात्याला पैसे देता
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# तुमच्याकडे की नाहीत — फक्त काम करते
# आपोआप होस्ट केलेली सेवा वापरते
# x402 द्वारे USDC ने प्रति भाषांतर पैसे द्या — साइनअप नको, खाते नको
bunx translator-agent -s ./dist -l all
```

समान कमांड. API की उपलब्ध असल्यास, ते तुमच्या प्रदात्यासह स्थानिकपणे चालते. नसल्यास, ते होस्ट केलेल्या API वर जाते आणि [x402](https://x402.org) द्वारे प्रति विनंती पैसे देते — HTTP 402 पेमेंट प्रोटोकॉल. तुमचा क्लायंट Base वर USDC देतो, भाषांतरे परत मिळवतो. ऑथ नको, विक्रेता नाते नको, इन्व्हॉइस नको.

Anthropic आणि OpenAI चे समर्थन. तुम्हाला जे मॉडेल हवे ते घ्या:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## प्रत्येक लिपी प्रणाली, हाताळली

हे टूल फक्त मजकूराचे भाषांतर करत नाही — त्याला माहीत आहे की प्रत्येक लेखन प्रणाली कशी रेंडर होते:

| लिपी | काय बदलते | का |
|---|---|---|
| **अरबी, हिब्रू, फारसी, उर्दू** | `dir="rtl"`, RTL फॉन्ट, 110% साईझ | अरबीला वाचनीय होण्यासाठी मोठे टाइप हवे; संपूर्ण लेआउट उलटे होते |
| **जपानी, चिनी, कोरियन** | CJK फॉन्ट स्टॅक, 1.8 line-height | अक्षरे निश्चित-रुंदी चौकोन आहेत; उभ्या श्वासाची जागा हवी |
| **हिंदी, बंगाली, तमिळ, तेलुगू** | भारतीय फॉन्ट, 1.8 line-height | शीर्षरेषा (शिरोरेखा) साठी अतिरिक्त उभी जागा हवी |
| **थाई** | `word-break: keep-all` | शब्दांमध्ये स्पेस नसतात — ब्राउझरला स्पष्ट लाइन-ब्रेक नियम हवेत |
| **बर्मी** | 2.2 line-height | कोणत्याही प्रमुख लिपीतील सर्वात उंच ग्लिफ्स |
| **खमेर** | 2.0 line-height | सबस्क्रिप्ट व्यंजन क्लस्टर्स उभ्या दिशेने स्टॅक होतात |

प्रति-लोकेल तयार केलेला CSS:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## कॅशिंग

भाषांतरे बिल्ड artifacts आहेत. बिल्ड टाइमवर तयार करा, आउटपुट कॅश करा, स्रोत न बदलल्यास स्किप करा.

### Vercel

Vercel आपोआप बिल्ड आउटपुट कॅश करते. `postbuild` जोडा आणि तुमचे काम पूर्ण:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

स्रोत न बदललेला = कॅश हिट = शून्य LLM कॉल = शून्य खर्च.

## पर्याय

```
Usage: translator-agent [options]

Options:
  -s, --source <path>      स्कॅन करण्यासाठी स्रोत डिरेक्टरी किंवा फाइल
  -l, --locales <locales>  लक्ष्य लोकेल, स्वल्पविरामांनी विभक्त किंवा 71 भाषांसाठी "all"
  -o, --output <path>      आउटपुट डिरेक्टरी (डिफॉल्ट: "./translations")
  -p, --provider <name>    anthropic | openai (डिफॉल्ट: "anthropic")
  -m, --model <id>         मॉडेल ओव्हरराइड
  -c, --concurrency <n>    कमाल समांतर LLM कॉल (डिफॉल्ट: 10)
  --api-url <url>          होस्ट केलेली सेवा URL (API की सेट नसल्यावर आपोआप वापरली)
```

| एक्स्टेंशन | रणनीती |
|---|---|
| `.json` | मूल्यांचे भाषांतर, की जपवा |
| `.md` / `.mdx` | मजकूराचे भाषांतर, सिंटॅक्स जपवा |
| `.html` / `.htm` | मजकूराचे भाषांतर, टॅग जपवा, `lang`/`dir` इंजेक्ट करा |
| बाकी सर्व | प्रत्येक लोकेल डिरेक्टरीमध्ये कॉपी करा |

### सर्व 71 लोकेल

`-l all` ~95% इंटरनेट वापरकर्त्यांना कव्हर करते: zh-CN, zh-TW, ja, ko, vi, th, id, ms, fil, my, hi, bn, ta, te, mr, gu, kn, ml, pa, ur, fa, tr, he, ar, kk, uz, fr, de, es, pt, pt-BR, it, nl, ca, gl, sv, da, no, fi, is, pl, cs, sk, hu, ro, bg, hr, sr, sl, uk, ru, lt, lv, et, el, ga, sw, am, ha, yo, zu, af, km, lo, ne, si, ka, az, mn

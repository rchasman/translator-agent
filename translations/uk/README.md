# translator-agent

Проблема локалізації за $10 000, вирішена за 90 секунд.

Компанії платять агенціям $0,10–0,25 за слово для локалізації своїх сайтів. Сайт з 5000 слів на 10 мов коштує $5000–12 000 і займає 2–4 тижні. Щоразу, коли ви змінюєте заголовок, лічильник запускається заново.

Цей інструмент робить це однією командою, на 71 мову, під час збирання:

```bash
bunx translator-agent -s ./dist -l all
```

Без агенцій. Без таблиць. Без прив'язки до постачальника. Без реєстрації. Ваші ключі, ваша збірка, ваші мови.

## Чому це працює

Переклад — розв'язана проблема. Локалізація — ні.

Google Translate може перетворити "Our hamsters are working on it" на японську. Але не може розпізнати, що жарт не спрацює в Японії, і замінити на щось, що спрацює — наприклад, посилання на команду інженерів, що працює всю ніч, що є і культурно доречним, і смішним у контексті.

Цей інструмент не перекладає. Він **транскреює** — той самий процес, за який рекламні агенції беруть $50 000 при адаптації кампанії на різних ринках. Тільки LLM уже знає всі культури, всі ідіоми, всі конвенції форматування. Він знає, що:

- `$49/month` стає `月額6,980円` в Японії — не "$49" зі значком єни зверху
- Сарказм вбиває англійською і вмирає у формальній японській
- "Drowning in paperwork" стає "noyade administrative" французькою — справжній французький вираз, а не дослівний переклад
- Німці залишають жарт про хом'ячків, бо Hamsterrad (хом'яче колесо) — справжня німецька ідіома
- Бразильцям потрібен невимушений регістр, інакше звучить, ніби це робот писав

Модель класифікує кожен рядок. UI-лейбли отримують прямий переклад. Маркетинговий контент адаптується. Гумор повністю перероблюється під цільову культуру.

## Що відбувається, коли ви запускаєте

Направте на вивід збірки. Він клонує всю структуру файлів для кожної локалі — перекладаючи текстові файли, копіюючи статичні ресурси та генеруючи все необхідне для деплою:

```
your-site/                          translations/
  index.html                          middleware.ts        ← визначення локалі
  about.html             →            _locales.css         ← типографіка для кожного скрипта
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", транскрейовано
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← скопійовано
                                        js/app.js          ← скопійовано
                                        images/logo.png    ← скопійовано
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% шрифт
                                        ...
                                      de/
                                        ...
```

Кожен HTML-файл отримує `lang` і `dir="rtl"`. Кожна локаль отримує CSS з правильним набором шрифтів, міжрядковим інтервалом та напрямком тексту. Генерується Vercel middleware, що читає `Accept-Language` і перенаправляє на відповідну локаль.

Деплойте на Vercel. Користувач з Токіо бачить японську з Hiragino Sans і міжрядковим інтервалом 1.8. Користувач з Каїра бачить RTL арабську з Noto Naskh розміром 110%. Користувач з Бангкока бачить тайську з `word-break: keep-all`, бо в тайській немає пробілів. Без конфігурації.

### У вашому пайплайні збірки

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

## Приносьте свої ключі або не приносьте

```bash
# У вас є ключі — працює локально, ви платите провайдеру LLM напряму
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# У вас немає ключів — просто працює
# Автоматично використовує хостований сервіс
# Платіть за переклад USDC через x402 — без реєстрації, без аккаунта
bunx translator-agent -s ./dist -l all
```

Та сама команда. Якщо є API-ключі, працює локально з вашим провайдером. Якщо немає, звертається до хостованого API і платить за запит через [x402](https://x402.org) — протокол платежів HTTP 402. Ваш клієнт платить USDC на Base, отримує переклади. Без авторизації, без відносин з постачальником, без рахунків.

Підтримує Anthropic і OpenAI. Приносьте будь-яку модель:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Всі системи письма, оброблені

Інструмент не просто перекладає текст — він знає, як рендериться кожна система письма:

| Скрипт | Що змінюється | Чому |
|---|---|---|
| **Арабська, іврит, фарсі, урду** | `dir="rtl"`, RTL шрифти, 110% розмір | Арабська потребує більшого шрифта для читабельності; весь макет відображається |
| **Японська, китайська, корейська** | CJK шрифтові стеки, 1.8 line-height | Символи — квадрати фіксованої ширини; потребують вертикального простору |
| **Хінді, бенгальська, тамільська, телугу** | Індійські шрифти, 1.8 line-height | Верхні риски (шірорекха) потребують додаткового вертикального простору |
| **Тайська** | `word-break: keep-all` | Немає пробілів між словами — браузеру потрібні явні правила переносу |
| **Бірманська** | 2.2 line-height | Найвищі гліфи з усіх основних скриптів |
| **Кхмерська** | 2.0 line-height | Підрядкові кластери приголосних складаються вертикально |

Згенерований CSS для кожної локалі:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Кешування

Переклади — це артефакти збірки. Генеруйте під час збірки, кешуйте вивід, пропускайте, коли джерело не змінилось.

### Vercel

Vercel автоматично кешує вивід збірки. Додайте `postbuild` і готово:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Джерело не змінилось = кеш попав = нуль викликів LLM = нуль коштів.

## Опції

```
Використання: translator-agent [опції]

Опції:
  -s, --source <path>      вихідна директорія або файл для сканування
  -l, --locales <locales>  цільові локалі, через кому або "all" для 71 мови
  -o, --output <path>      директорія виводу (за замовчуванням: "./translations")
  -p, --provider <name>    anthropic | openai (за замовчуванням: "anthropic")
  -m, --model <id>         перевизначення моделі
  -c, --concurrency <n>    максимум паралельних викликів LLM (за замовчуванням: 10)
  --api-url <url>          URL хостованого сервісу (автовикористання, коли немає API-ключів)
```

| Розширення | Стратегія |
|---|---|
| `.json` | Перекладати значення, зберігати ключі |
| `.md` / `.mdx` | Перекладати текст, зберігати синтаксис |
| `.html` / `.htm` | Перекладати текст, зберігати теги, вставляти `lang`/`dir` |
| Все інше | Копіювати в директорію кожної локалі |

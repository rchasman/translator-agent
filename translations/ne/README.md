# translator-agent

$10,000 को स्थानीयकरण समस्या, 90 सेकेण्डमा समाधान।

कम्पनीहरूले आफ्ना साइटहरूको स्थानीयकरण गर्न एजेन्सीहरूलाई प्रति शब्द $0.10–0.25 तिर्छन्। 5,000 शब्दको साइटलाई 10 भाषाहरूमा अनुवाद गर्न $5,000–12,000 खर्च हुन्छ र 2–4 हप्ता लाग्छ। हरेकपटक तपाईंले शीर्षक परिवर्तन गर्दा, मिटर फेरि सुरु हुन्छ।

यो उपकरणले एउटै कमाण्डमा, 71 भाषाहरूमा, तपाईंको बिल्ड चरणमा नै गर्छ:

```bash
bunx translator-agent -s ./dist -l all
```

कुनै एजेन्सी छैन। कुनै स्प्रेडसिटहरू छैनन्। कुनै भेन्डर लक-इन छैन। कुनै साइन अप छैन। तपाईंको कुञ्जी, तपाईंको बिल्ड, तपाईंको भाषाहरू।

> **तपाईं प्रमाण पढ्दै हुनुहुन्छ।** यो README `bunx translator-agent -s README.md -l all` चलाएर अनुवाद गरिएको हो। [जापानी संस्करण](./translations/ja/README.md) पढ्नुहोस् — यसले केवल "the meter restarts" लाई अनुवाद मात्र गरेको छैन, यसलाई जापानी व्यावसायिक मुहावराले बदलेको छ। [जर्मन संस्करण](./translations/de/README.md) 30% लामो छ किनभने जर्मन सधैं त्यस्तै हुन्छ। [अरबी संस्करण](./translations/ar/README.md) दायाँबाट देब्रे पढिन्छ। [ब्राजिलियन पोर्तुगाली संस्करण](./translations/pt-BR/README.md) ब्राजिलियनले लेखेको जस्तै लाग्छ, किनभने त्यही मुख्य कुरा हो।

## यसले किन काम गर्छ

अनुवाद समाधान भएको समस्या हो। स्थानीयकरण होइन।

Google Translate ले "Our hamsters are working on it" लाई जापानीमा बदल्न सक्छ। यसले गर्न नसक्ने कुरा भनेको यो मजाक जापानमा काम नगर्ने कुरा चिन्ने र यसलाई काम गर्ने कुराले बदल्नु हो — जस्तै इञ्जिनियरिङ टीमले रातभर काम गरिरहेको कुरा भन्नु, जुन सांस्कृतिक रूपमा उपयुक्त र सन्दर्भमा मजाकिया दुवै छ।

यो उपकरणले अनुवाद गर्दैन। यसले **ट्रान्सक्रिएट** गर्छ — त्यहि प्रक्रिया जसको लागि विज्ञापन एजेन्सीहरूले बजारहरूमा अभियान अनुकूलन गर्दा $50,000 लिन्छन्। केवल LLM ले पहिले नै हरेक संस्कृति, हरेक मुहावरा, हरेक ढाँचा परम्परा जान्दछ। यसलाई थाहा छ कि:

- `$49/month` जापानमा `月額6,980円` बन्छ — "$49" मा येन चिन्ह टाँसेको होइन
- व्यंग्य अङ्ग्रेजीमा मार्छ र औपचारिक जापानीमा मर्छ
- "Drowning in paperwork" फ्रेन्चमा "noyade administrative" बन्छ — वास्तविक फ्रेन्च अभिव्यक्ति, शब्द-दर-शब्द अनुवाद होइन
- जर्मनहरूले ह्यामस्टर जोक राख्छन् किनभने Hamsterrad (hamster wheel) वास्तविक जर्मन मुहावरा हो
- ब्राजिलियनहरूलाई अनौपचारिक शैली चाहिन्छ नत्र रोबोटले लेखेको जस्तै लाग्छ

मोडेलले प्रत्येक स्ट्रिङलाई वर्गीकरण गर्छ। UI लेबलहरूले प्रत्यक्ष अनुवाद पाउँछन्। मार्केटिङ कपी अनुकूलित हुन्छ। हास्य लक्षित संस्कृतिको लागि पूर्ण रूपमा पुनः निर्मित हुन्छ।

## तपाईंले चलाउँदा के हुन्छ

यसलाई तपाईंको बिल्ड आउटपुटमा देखाउनुहोस्। यसले प्रत्येक लोकेलको लागि सम्पूर्ण फाइल ट्री क्लोन गर्छ — पाठ फाइलहरू अनुवाद गर्दै, स्थिर सम्पत्तिहरू प्रतिलिपि गर्दै, र deployment को लागि आवश्यक सबै कुरा उत्पादन गर्दै:

```
your-site/                          translations/
  index.html                          middleware.ts        ← लोकेल पहिचान
  about.html             →            _locales.css         ← प्रत्येक लिपिको लागि टाइपोग्राफी
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ट्रान्सक्रिएट गरिएको
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← प्रतिलिपि गरिएको
                                        js/app.js          ← प्रतिलिपि गरिएको
                                        images/logo.png    ← प्रतिलिपि गरिएको
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% फन्ट
                                        ...
                                      de/
                                        ...
```

प्रत्येक HTML फाइलले `lang` र `dir="rtl"` इन्जेक्ट गराउँछ। प्रत्येक लोकेलले सही फन्ट स्ट्याक, लाइन-हाइट, र पाठ दिशा सहित CSS पाउँछ। Vercel middleware उत्पादन गरिन्छ जसले `Accept-Language` पढ्छ र सही लोकेलमा पुनः लेख्छ।

Vercel मा deploy गर्नुहोस्। टोकियोको प्रयोगकर्ताले 1.8 लाइन-हाइटमा Hiragino Sans सहित जापानी देख्छ। काइरोको प्रयोगकर्ताले 110% साइजमा Noto Naskh सहित RTL अरबी देख्छ। बैंककको प्रयोगकर्ताले `word-break: keep-all` सहित थाई देख्छ किनभने थाईमा खाली ठाउँहरू हुँदैनन्। कुनै कन्फिग छैन।

## 90 सेकेण्ड, 4 हप्ता होइन

```bash
# तीन भाषाहरू, एउटा JSON फाइल
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
सम्पन्न। 9.5s मा 3 फाइलहरू लेखिएका

# तपाईंको सम्पूर्ण साइट, पृथ्वीका हरेक भाषा
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
सम्पन्न। 94s मा 142 फाइलहरू अनुवाद गरिएका, 284 स्थिर फाइलहरू प्रतिलिपि गरिएका
```

### तपाईंको बिल्ड पाइपलाइनमा

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

प्रत्येक deploy 71 भाषाहरूमा पठाइन्छ। अनुवादहरू बिल्ड artifacts हुन् — cache गरिएका, स्रोत परिवर्तन भएमा मात्र पुनः उत्पादित।

## आफ्नै कुञ्जी ल्याउनुहोस् वा नगर्नुहोस्

```bash
# तपाईंसँग कुञ्जीहरू छन् — स्थानीय रूपमा चल्छ, तपाईं आफ्नो LLM प्रदायकलाई प्रत्यक्ष तिर्नुहुन्छ
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# तपाईंसँग कुञ्जीहरू छैनन् — बस काम गर्छ
# hosted service स्वचालित रूपमा प्रयोग गर्छ
# x402 मार्फत USDC सहित प्रति अनुवाद तिर्नुहोस् — कुनै साइन अप छैन, कुनै खाता छैन
bunx translator-agent -s ./dist -l all
```

उही कमाण्ड। यदि API कुञ्जीहरू छन् भने, यो तपाईंको प्रदायकसँग स्थानीय रूपमा चल्छ। यदि छैन भने, यो hosted API मा हिट गर्छ र [x402](https://x402.org) मार्फत प्रति अनुरोध तिर्छ — HTTP 402 भुक्तानी प्रोटोकल। तपाईंको क्लाइन्टले Base मा USDC तिर्छ, अनुवादहरू फिर्ता पाउँछ। कुनै प्रमाणीकरण छैन, कुनै भेन्डर सम्बन्ध छैन, कुनै बीजकहरू छैनन्।

Anthropic र OpenAI लाई समर्थन गर्छ। तपाईंले चाहेको कुनै पनि मोडेल ल्याउनुहोस्:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## हरेक लिपि प्रणाली, व्यवस्थापन गरिएको

उपकरणले पाठ अनुवाद मात्र गर्दैन — यसलाई प्रत्येक लेखन प्रणाली कसरी रेन्डर हुन्छ भन्ने थाहा छ:

| लिपि | के परिवर्तन हुन्छ | किन |
|---|---|---|
| **अरबी, हिब्रू, फारसी, उर्दू** | `dir="rtl"`, RTL फन्टहरू, 110% साइज | अरबीलाई पढ्न मिल्ने बनाउन ठूलो टाइप चाहिन्छ; सम्पूर्ण लेआउट उल्टिन्छ |
| **जापानी, चिनियाँ, कोरियाली** | CJK फन्ट स्ट्याकहरू, 1.8 line-height | अक्षरहरू निश्चित-चौडाइ वर्गहरू हुन्; ठाडो सास फेर्ने ठाउँ चाहिन्छ |
| **हिन्दी, बंगाली, तमिल, तेलुगु** | इन्डिक फन्टहरू, 1.8 line-height | शिरोरेखा (headstrokes) लाई अतिरिक्त ठाडो ठाउँ चाहिन्छ |
| **थाई** | `word-break: keep-all` | शब्दहरू बीच खाली ठाउँ हुँदैन — ब्राउजरलाई स्पष्ट line-break नियमहरू चाहिन्छ |
| **बर्मिज** | 2.2 line-height | कुनै पनि प्रमुख लिपिको सबैभन्दा अग्लो ग्लिफहरू |
| **खमेर** | 2.0 line-height | सबस्क्रिप्ट व्यञ्जन क्लस्टरहरू ठाडो रूपमा स्ट्याक हुन्छन् |

प्रति-लोकेल उत्पादित CSS:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Cache गर्दै

अनुवादहरू बिल्ड artifacts हुन्। बिल्ड समयमा उत्पादन गर्नुहोस्, आउटपुट cache गर्नुहोस्, स्रोत परिवर्तन नभएको बेला छोड्नुहोस्।

### Vercel

Vercel ले बिल्ड आउटपुट स्वचालित रूपमा cache गर्छ। `postbuild` थप्नुहोस् र तपाईंको काम सकियो:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

स्रोत अपरिवर्तित = cache hit = शून्य LLM कलहरू = शून्य लागत।

## विकल्पहरू

```
प्रयोग: translator-agent [विकल्पहरू]

विकल्पहरू:
  -s, --source <path>      स्क्यान गर्न स्रोत डाइरेक्टरी वा फाइल
  -l, --locales <locales>  लक्षित लोकेलहरू, अल्पविराम-विभाजित वा "all" 71 भाषाहरूको लागि
  -o, --output <path>      आउटपुट डाइरेक्टरी (पूर्वनिर्धारित: "./translations")
  -p, --provider <name>    anthropic | openai (पूर्वनिर्धारित: "anthropic")
  -m, --model <id>         मोडेल ओभरराइड
  -c, --concurrency <n>    अधिकतम समानान्तर LLM कलहरू (पूर्वनिर्धारित: 10)
  --api-url <url>          hosted service URL (कुनै API कुञ्जीहरू सेट नभएको बेला स्वतः प्रयोग हुन्छ)
```

| एक्सटेन्सन | रणनीति |
|---|---|
| `.json` | मानहरू अनुवाद गर्नुहोस्, कुञ्जीहरू संरक्षण गर्नुहोस् |
| `.md` / `.mdx` | पाठ अनुवाद गर्नुहोस्, syntax संरक्षण गर्नुहोस् |
| `.html` / `.htm` | पाठ अनुवाद गर्नुहोस्, tags संरक्षण गर्नुहोस्, `lang`/`dir` inject गर्नुहोस् |
| बाँकी सबै | प्रत्येक लोकेल डाइरेक्टरीमा प्रतिलिपि गर्नुहोस् |

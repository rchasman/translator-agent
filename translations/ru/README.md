# translator-agent

Проблема локализации за $10 000, решённая за 90 секунд.

Компании платят агентствам $0,10–0,25 за слово за локализацию сайтов. Сайт в 5000 слов на 10 языков обходится в $5000–12 000 и занимает 2–4 недели. Каждый раз, когда вы меняете заголовок, счётчик запускается заново.

Этот инструмент делает это одной командой, на 71 язык, прямо во время сборки:

```bash
bunx translator-agent -s ./dist -l all
```

Никаких агентств. Никаких таблиц. Никакой привязки к поставщикам. Никакой регистрации. Ваши ключи, ваша сборка, ваши языки.

## Почему это работает

Перевод — решённая задача. Локализация — нет.

Google Translate может перевести "Our hamsters are working on it" на японский. Но он не может понять, что шутка не зайдёт в Японии, и заменить её на что-то подходящее — например, про команду разработчиков, работающих всю ночь, что одновременно культурно уместно и смешно в контексте.

Этот инструмент не переводит. Он делает **транскреацию** — тот же процесс, за который рекламные агентства берут $50 000 при адаптации кампании для разных рынков. Только языковая модель уже знает все культуры, все идиомы, все форматы. Она знает, что:

- `$49/month` становится `月額6,980円` в Японии — не "$49" со знаком йены налепленным сверху
- Сарказм убивает на английском и умирает в формальном японском
- "Drowning in paperwork" становится "noyade administrative" на французском — настоящим французским выражением, не дословным переводом
- Немцы оставляют шутку про хомячков, потому что Hamsterrad (беличье колесо) — настоящая немецкая идиома
- Бразильцам нужен непринуждённый регистр, иначе будет звучать как робот

Модель классифицирует каждую строку. UI-элементы получают прямой перевод. Маркетинговые тексты адаптируются. Юмор полностью пересоздаётся для целевой культуры.

## Что происходит при запуске

Направляете её на результат сборки. Она клонирует всё дерево файлов для каждой локали — переводит текстовые файлы, копирует статические ресурсы и генерирует всё необходимое для развёртывания:

```
your-site/                          translations/
  index.html                          middleware.ts        ← определение локали
  about.html             →            _locales.css         ← типографика для письменностей
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", транскреация
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← скопирован
                                        js/app.js          ← скопирован
                                        images/logo.png    ← скопирован
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% шрифт
                                        ...
                                      de/
                                        ...
```

В каждый HTML-файл добавляются `lang` и `dir="rtl"`. Каждая локаль получает CSS с правильным стеком шрифтов, межстрочным интервалом и направлением текста. Генерируется middleware для Vercel, который читает `Accept-Language` и перенаправляет на нужную локаль.

Развёртывание на Vercel. Пользователь в Токио видит японский с Hiragino Sans и межстрочным интервалом 1.8. Пользователь в Каире видит арабский справа налево с Noto Naskh размером 110%. Пользователь в Бангкоке видит тайский с `word-break: keep-all`, потому что в тайском нет пробелов. Никаких настроек.

## 90 секунд, не 4 недели

```bash
# Три языка, один JSON файл
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
Готово. 3 файла записано за 9,5с

# Весь ваш сайт, все языки планеты
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
Готово. 142 файла переведено, 284 статических файла скопировано за 94с
```

### В вашем пайплайне сборки

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

Каждое развёртывание поставляется на 71 языке. Переводы — это артефакты сборки, кешируются и регенерируются только при изменении исходников.

## Свои ключи или без них

```bash
# У вас есть ключи — работает локально, платите провайдеру LLM напрямую
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# Нет ключей — просто работает
# Автоматически использует хостинговый сервис
# Платите за перевод USDC через x402 — без регистрации, без аккаунта
bunx translator-agent -s ./dist -l all
```

Та же команда. Если есть API-ключи, работает локально с вашим провайдером. Если нет — обращается к хостинговому API и платит за запрос через [x402](https://x402.org) — протокол HTTP 402 платежей. Ваш клиент платит USDC на Base, получает переводы обратно. Без авторизации, без отношений с поставщиками, без счетов.

Поддерживает Anthropic и OpenAI. Используйте любую модель:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Все системы письменности под контролем

Инструмент не просто переводит текст — он знает, как отображается каждая система письменности:

| Письменность | Что меняется | Зачем |
|---|---|---|
| **Арабская, иврит, фарси, урду** | `dir="rtl"`, RTL шрифты, 110% размер | Арабскому нужен крупнее шрифт для читаемости; вся раскладка зеркальная |
| **Японская, китайская, корейская** | CJK стеки шрифтов, 1.8 line-height | Символы — квадраты фиксированной ширины; нужно вертикальное пространство |
| **Хинди, бенгали, тамильская, телугу** | Индийские шрифты, 1.8 line-height | Верхним штрихам (широрекха) нужно дополнительное вертикальное место |
| **Тайская** | `word-break: keep-all` | Нет пробелов между словами — браузеру нужны явные правила переноса |
| **Бирманская** | 2.2 line-height | Самые высокие глифы среди основных письменностей |
| **Кхмерская** | 2.0 line-height | Подстрочные согласные кластеры складываются вертикально |

Генерируемый CSS для локали:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Кеширование

Переводы — это артефакты сборки. Генерируются во время сборки, кешируется вывод, пропускается при неизменённых исходниках.

### Vercel

Vercel кеширует вывод сборки автоматически. Добавьте `postbuild` и всё готово:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Исходники не изменились = попадание в кеш = ноль вызовов LLM = ноль затрат.

## Опции

```
Использование: translator-agent [опции]

Опции:
  -s, --source <путь>      исходная директория или файл для сканирования
  -l, --locales <локали>   целевые локали через запятую или "all" для 71 языка
  -o, --output <путь>      выходная директория (по умолчанию: "./translations")
  -p, --provider <имя>     anthropic | openai (по умолчанию: "anthropic")
  -m, --model <id>         переопределение модели
  -c, --concurrency <n>    макс. параллельных вызовов LLM (по умолчанию: 10)
  --api-url <url>          URL хостингового сервиса (используется авто при отсутствии API ключей)
```

| Расширение | Стратегия |
|---|---|
| `.json` | Переводить значения, сохранять ключи |
| `.md` / `.mdx` | Переводить текст, сохранять синтаксис |
| `.html` / `.htm` | Переводить текст, сохранять теги, добавлять `lang`/`dir` |
| Все остальные | Копировать в каждую директорию локали |

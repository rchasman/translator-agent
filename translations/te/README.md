₹7,50,000 లోకలైజేషన్ సమస్య, 90 సెకన్లలో పరిష్కరం.

కంపెనీలు తమ సైట్లను లోకలైజ్ చేయడానికి ఏజెన్సీలకు పదానికి ₹8-20 చెల్లిస్తున్నారు. 5,000 పదాల సైట్ను 10 భాషల్లోకి అనువదించడానికి ₹4,00,000-10,00,000 ఖర్చు మరియు 2-4 వారాలు సమయం పడుతుంది. మీరు ఒక హెడ్‌లైన్ మార్చిన ప్రతిసారీ, మీటర్ మళ్లీ మొదలు కావాలి.

ఈ సాధనం ఒకే కమాండ్‌తో 71 భాషల్లోకి మీ బిల్డ్ స్టెప్‌లోనే చేసేస్తుంది:

```bash
bunx translator-agent -s ./dist -l all
```

## ఇది ఎందుకు పని చేస్తుంది

అనువాదం పరిష్కరించబడిన సమస్య. లోకలైజేషన్ కాదు.

Google Translate "మా గుండెల్లు దానిపై పని చేస్తున్నాయి" అని జపనీస్‌లోకి మార్చగలదు. కానీ ఆ జోక్ జపాన్‌లో నవ్వించదని గుర్తించి, దానికి బదులుగా నవ్వించే మరో దాన్ని వాడలేదు — ఇంజనీరింగ్ టీం రాత్రంతా మేలుకుని పని చేయడం గురించి, ఇది సాంస్కృతికంగా సరైనది మరియు ఆ సందర్భంలో హాస్యాస్పదం కూడా.

ఈ సాధనం అనువదించదు. ఇది **ట్రాన్స్‌క్రియేట్** చేస్తుంది — ప్రచార కంపెనీలను మార్కెట్లలో స్వీకరించేటప్పుడు యాడ్ ఏజెన్సీలు ₹40,00,000 వసూలు చేసే అదే ప్రక్రియ. తేడా ఏంటంటే LLM కు ఇప్పటికే ప్రతి సంస్కృతి, ప్రతి భాషా వైశిష్ట్యం, ప్రతి ఫార్మేటింగ్ కన్వెన్షన్ తెలుసు. దానికి తెలుసు:

- `₹49/నెల` జపాన్‌లో `月額6,980円` అవుతుందని — కేవలం "₹49" మీద యెన్ చిహ్నం అంటుకోవడం కాదు
- వ్యంగ్యం ఆంగ్లంలో బాగుంటుందని, అధికారిక జపనీస్‌లో చనిపోతుందని
- "Drowning in paperwork" అనేది ఫ్రెంచ్‌లో "noyade administrative" అవుతుందని — నిజమైన ఫ్రెంచ్ వ్యక్తీకరణ, పదానికిపదం అనువాదం కాదు
- జర్మన్లు గుండెల్ల జోక్‌ను ఉంచుకుంటారని ఎందుకంటే Hamsterrad (గుండెల్ చక్రం) నిజమైన జర్మన్ భాషా వైశిష్ట్యం
- బ్రెజిలియన్లకు సాధారణ రిజిస్టర్ అవసరం లేకుంటే రోబోట్ వ్రాసినట్లు అనిపిస్తుంది

మోడల్ ప్రతి స్ట్రింగ్‌ను వర్గీకరిస్తుంది. UI లేబుల్స్ ప్రత్యక్ష అనువాదం పొందుతాయి. మార్కెటింగ్ కాపీ స్వీకరించబడుతుంది. హాస్యం లక్ష్య సంస్కృతికి పూర్తిగా మార్చబడుతుంది.

## మీరు దీన్ని రన్ చేసినప్పుడు ఏం జరుగుతుంది

మీ బిల్డ్ అవుట్‌పుట్ వైపు చూపించండి. ఇది ప్రతి లొకేల్‌కు మొత్తం ఫైల్ ట్రీని క్లోన్ చేస్తుంది — టెక్స్ట్ ఫైల్స్ అనువదిస్తుంది, స్టాటిక్ అసెట్లు కాపీ చేస్తుంది, మరియు డిప్లాయ్‌మెంట్‌కు అవసరమైన అన్నింటినీ జనరేట్ చేస్తుంది:

```
your-site/                          translations/
  index.html                          middleware.ts        ← లొకేల్ గుర్తింపు
  about.html             →            _locales.css         ← స్క్రిప్ట్ ఆధారంగా టైపోగ్రఫీ
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ట్రాన్స్‌క్రియేట్ చేయబడింది
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← కాపీ చేయబడింది
                                        js/app.js          ← కాపీ చేయబడింది
                                        images/logo.png    ← కాపీ చేయబడింది
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% ఫాంట్
                                        ...
                                      de/
                                        ...
```

ప్రతి HTML ఫైల్‌కు `lang` మరియు `dir="rtl"` ఇంజెక్ట్ అవుతుంది. ప్రతి లొకేల్‌కు సరైన ఫాంట్ స్టాక్, లైన్-హైట్, మరియు టెక్స్ట్ దిశతో CSS లభిస్తుంది. Vercel మిడిల్‌వేర్ జనరేట్ అవుతుంది, అది `Accept-Language` చదివి సరైన లొకేల్‌కు రీరైట్ చేస్తుంది.

Vercel కు డిప్లాయ్ చేయండి. టోక్యోలో ఉన్న వినియోగదారు 1.8 లైన్-హైట్‌తో Hiragino Sans లో జపనీస్ చూస్తారు. కైరోలో ఉన్న వినియోగదారు 110% పరిమాణంలో Noto Naskh తో RTL అరబిక్ చూస్తారు. బ్యాంకాక్‌లో ఉన్న వినియోగదారు `word-break: keep-all` తో థాయి చూస్తారు ఎందుకంటే థాయిలో ఖాళీలు ఉండవు. కాన్ఫిగ్ అవసరం లేదు.

## మీ కీలు తీసుకురండి లేదా వదిలేయండి

```bash
# మీకు కీలు ఉన్నాయి — లోకల్‌గా రన్ అవుతుంది, మీరు మీ LLM ప్రొవైడర్‌కు ప్రత్యక్షంగా చెల్లిస్తారు
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# మీకు కీలు లేవు — అలానే పని చేస్తుంది
# హోస్ట్ చేసిన సేవను స్వయంచాలకంగా వాడుతుంది
# x402 ద్వారా USDC తో అనువాదానికి చెల్లించండి — సైన్‌అప్ లేదు, అకౌంట్ లేదు
bunx translator-agent -s ./dist -l all
```

అదే కమాండ్. API కీలు ఉంటే, మీ ప్రొవైడర్‌తో లోకల్‌గా రన్ అవుతుంది. లేకుంటే, హోస్ట్ చేసిన API ని హిట్ చేసి [x402](https://x402.org) ద్వారా రిక్వెస్ట్‌కు చెల్లిస్తుంది — HTTP 402 పేమెంట్ ప్రొటోకాల్. మీ క్లైంట్ Base పై USDC చెల్లించి, అనువాదాలు తిరిగి పొందుతుంది. అథ్ లేదు, వెండర్ సంబంధం లేదు, ఇన్‌వాయిస్‌లు లేవు.

Anthropic మరియు OpenAI ను సపోర్ట్ చేస్తుంది. మీకు కావాలసిన మోడల్ తీసుకురండి:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## ప్రతి స్క్రిప్ట్ వ్యవస్థ, హ్యాండిల్ చేయబడుతుంది

ఈ సాధనం కేవలం టెక్స్ట్ అనువదించడమే కాదు — ప్రతి రైటింగ్ సిస్టమ్ ఎలా రెండర్ అవుతుందో కూడా తెలుసు:

| స్క్రిప్ట్ | ఏమి మారుతుంది | ఎందుకు |
|---|---|---|
| **అరబిక్, హీబ్రూ, ఫార్సీ, ఉర్దూ** | `dir="rtl"`, RTL ఫాంట్లు, 110% పరిమాణం | అరబిక్‌కు పఠనీయంగా ఉండటానికి పెద్ద టైప్ కావాలి; మొత్తం లేఅవుట్ అద్దంగా మారుతుంది |
| **జపనీస్, చైనీస్, కొరియన్** | CJK ఫాంట్ స్టాక్లు, 1.8 లైన్-హైట్ | అక్షరాలు స్థిర-వెడల్పు చతురస్రాలు; నిలువు ఉపశ్వాస స్థలం కావాలి |
| **హిందీ, బెంగాలీ, తమిళ్, తెలుగు** | ఇండిక్ ఫాంట్లు, 1.8 లైన్-హైట్ | హెడ్‌స్ట్రోక్‌లకు (శిరోరేఖ) అదనపు నిలువు స్థలం కావాలి |
| **థాయి** | `word-break: keep-all` | పదాల మధ్య ఖాళీలు లేవు — బ్రౌజర్‌కు స్పష్టమైన లైన్-బ్రేక్ నియమాలు కావాలి |
| **బర్మీస్** | 2.2 లైన్-హైట్ | ప్రధాన స్క్రిప్ట్‌లలో అత్యంత ఎత్తైన గ్లిఫ్‌లు |
| **ఖ్మేర్** | 2.0 లైన్-హైట్ | సబ్‌స్క్రిప్ట్ కాన్సొనెంట్ క్లస్టర్లు నిలువుగా పేర్చబడతాయి |

ప్రతి లొకేల్ కోసం జనరేట్ చేసిన CSS:

/* ar/_locale.css */

  line-height: 1.9;
  font-size: 110%;
}

అనువాదాలు బిల్డ్ ఆర్టిఫాక్ట్లు. బిల్డ్ టైమ్‌లో జనరేట్ చేయండి, అవుట్‌పుట్ క్యాష్ చేయండి, సోర్స్ మారకపోతే స్కిప్ చేయండి.

### Vercel

Vercel స్వయంచాలకంగా బిల్డ్ అవుట్‌పుట్ క్యాష్ చేస్తుంది. `postbuild` జోడించండి మరియు పూర్తి:

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

సోర్స్ మారకపోతే = క్యాష్ హిట్ = LLM కాల్స్ లేవు = ఖర్చు లేదు.

## ఎంపికలు

```
వాడుక: translator-agent [options]

ఎంపికలు:
  -s, --source <path>      స్కాన్ చేయాల సోర్స్ డైరెక్టరీ లేదా ఫైల్
  -l, --locales <locales>  లక్ష్య లొకేల్స్, కామాతో వేరు చేసినవి లేదా 71 భాషలకు "all"
  -o, --output <path>      అవుట్‌పుట్ డైరెక్టరీ (డిఫాల్ట్: "./translations")
  -p, --provider <name>    anthropic | openai (డిఫాల్ట్: "anthropic")
  -m, --model <id>         మోడల్ ఓవర్‌రైడ్
  -c, --concurrency <n>    గరిష్ట సమాంతర LLM కాల్స్ (డిఫాల్ట్: 10)
  --api-url <url>          హోస్ట్ చేసిన సేవ URL (API కీలు సెట్ చేయకపోతే స్వయం-వాడుక)
```

| ఎక్స్‌టెన్షన్ | వ్యూహం |
|---|---|
| `.json` | విలువలను అనువదించు, కీలను కాపాడు |
| `.md` / `.mdx` | టెక్స్ట్ అనువదించు, సింట్యాక్స్ కాపాడు |
| `.html` / `.htm` | టెక్స్ట్ అనువదించు, టాగ్‌లను కాపాడు, `lang`/`dir` ఇంజెక్ట్ చేయు |
| మిగతావన్నీ | ప్రతి లొకేల్ డైరెక్టరీలోకి కాపీ చేయు |

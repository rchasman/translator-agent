## ਇਹ ਕਿਉਂ ਕੰਮ ਕਰਦਾ ਹੈ

ਅਨੁਵਾਦ ਤਾਂ ਹੱਲ ਹੋ ਗਿਆ ਹੈ। ਸਥਾਨੀਕਰਨ ਨਹੀਂ।

ਗੂਗਲ ਅਨੁਵਾਦ "Our hamsters are working on it" ਨੂੰ ਜਪਾਨੀ ਵਿੱਚ ਬਦਲ ਸਕਦਾ ਹੈ। ਜੋ ਇਹ ਨਹੀਂ ਕਰ ਸਕਦਾ ਉਹ ਇਹ ਪਛਾਣਨਾ ਹੈ ਕਿ ਇਹ ਚੁਟਕੁਲਾ ਜਪਾਨ ਵਿੱਚ ਨਹੀਂ ਜੰਚਦਾ, ਅਤੇ ਇਸਦੀ ਥਾਂ ਕੁਝ ਅਜਿਹਾ ਲਗਾਉਣਾ ਜੋ ਸਫਲ ਹੋਵੇ — ਜਿਵੇਂ ਇੰਜੀਨੀਅਰਿੰਗ ਟੀਮ ਦੇ ਰਾਤੋਂ ਰਾਤ ਮਿਹਨਤ ਕਰਨ ਦਾ ਹਵਾਲਾ, ਜੋ ਸੱਭਿਆਚਾਰਕ ਤੌਰ 'ਤੇ ਢੁਕਵਾਂ ਅਤੇ ਸੰਦਰਭ ਵਿੱਚ ਮਜ਼ੇਦਾਰ ਹੈ।

ਇਹ ਸੰਦ ਅਨੁਵਾਦ ਨਹੀਂ ਕਰਦਾ। ਇਹ **ਪੁਨਰ-ਰਚਨਾ** ਕਰਦਾ ਹੈ — ਉਹੀ ਪ੍ਰਕਿਰਿਆ ਜਿਸ ਲਈ ਇਸ਼ਤਿਹਾਰੀ ਏਜੰਸੀਆਂ ਮਾਰਕੀਟਾਂ ਵਿੱਚ ਮੁਹਿੰਮ ਅਪਣਾਉਣ ਲਈ ₹40 ਲੱਖ ਲੈਂਦੀਆਂ ਹਨ। ਸਿਵਾਏ ਇਸਦੇ ਕਿ LLM ਪਹਿਲਾਂ ਹੀ ਹਰ ਸੱਭਿਆਚਾਰ, ਹਰ ਮੁਹਾਵਰੇ, ਹਰ ਫਾਰਮੈਟਿੰਗ ਨਿਯਮ ਨੂੰ ਜਾਣਦਾ ਹੈ। ਇਹ ਜਾਣਦਾ ਹੈ ਕਿ:

- `$49/month` ਜਪਾਨ ਵਿੱਚ `月額6,980円` ਬਣਦਾ ਹੈ — ਯੇਨ ਦਾ ਚਿੰਨ੍ਹ ਲਗਾ ਕੇ "$49" ਨਹੀਂ
- ਵਿਅੰਗ ਅੰਗਰੇਜ਼ੀ ਵਿੱਚ ਮਾਰਦਾ ਹੈ ਅਤੇ ਰਸਮੀ ਜਪਾਨੀ ਵਿੱਚ ਮਰ ਜਾਂਦਾ ਹੈ
- "Drowning in paperwork" ਫ੍ਰੈਂਚ ਵਿੱਚ "noyade administrative" ਬਣਦਾ ਹੈ — ਇੱਕ ਅਸਲੀ ਫ੍ਰੈਂਚ ਪਦ, ਸ਼ਬਦ-ਦਰ-ਸ਼ਬਦ ਅਨੁਵਾਦ ਨਹੀਂ
- ਜਰਮਨ ਭਾਸ਼ੀ ਹੈਂਮਸਟਰ ਦਾ ਚੁਟਕੁਲਾ ਰੱਖਦੇ ਹਨ ਕਿਉਂਕਿ Hamsterrad (ਹੈਂਮਸਟਰ ਵੀਲ) ਅਸਲ ਜਰਮਨ ਮੁਹਾਵਰਾ ਹੈ
- ਬ੍ਰਾਜ਼ੀਲੀਅਨਾਂ ਨੂੰ ਆਮ ਰਜਿਸਟਰ ਚਾਹੀਦਾ ਹੈ ਨਹੀਂ ਤਾਂ ਲੱਗਦਾ ਹੈ ਜਿਵੇਂ ਰੋਬੋਟ ਨੇ ਲਿਖਿਆ ਹੋਵੇ

ਮਾਡਲ ਹਰ ਸਤਰ ਦਾ ਵਰਗੀਕਰਨ ਕਰਦਾ ਹੈ। UI ਲੇਬਲਾਂ ਦਾ ਸਿੱਧਾ ਅਨੁਵਾਦ ਹੁੰਦਾ ਹੈ। ਮਾਰਕੀਟਿੰਗ ਕਾਪੀ ਨੂੰ ਢਾਲਿਆ ਜਾਂਦਾ ਹੈ। ਮਜ਼ਾਕ ਨੂੰ ਨਿਸ਼ਾਨਾ ਸੱਭਿਆਚਾਰ ਲਈ ਪੂਰੀ ਤਰ੍ਹਾਂ ਨਵੇਂ ਸਿਰੇ ਤੋਂ ਬਣਾਇਆ ਜਾਂਦਾ ਹੈ।

## ਜਦੋਂ ਤੁਸੀਂ ਇਸਨੂੰ ਚਲਾਉਂਦੇ ਹੋ ਤਾਂ ਕੀ ਹੁੰਦਾ ਹੈ

ਇਸਨੂੰ ਆਪਣੇ ਬਿਲਡ ਆਉਟਪੁੱਟ 'ਤੇ ਇਸ਼ਾਰਾ ਕਰੋ। ਇਹ ਹਰ ਸਥਾਨ ਲਈ ਸਾਰਾ ਫ਼ਾਈਲ ਢਾਂਚਾ ਕਲੋਨ ਕਰਦਾ ਹੈ — ਟੈਕਸਟ ਫ਼ਾਈਲਾਂ ਦਾ ਅਨੁਵਾਦ ਕਰਦਾ, ਸਥਿਰ ਸੰਪੱਤੀਆਂ ਕਾਪੀ ਕਰਦਾ, ਅਤੇ ਤੈਨਾਤੀ ਲਈ ਲੋੜੀਂਦੀ ਸਭ ਕੁਝ ਤਿਆਰ ਕਰਦਾ:

```
your-site/                          translations/
  index.html                          middleware.ts        ← ਸਥਾਨ ਖੋਜ
  about.html             →            _locales.css         ← ਸਕ੍ਰਿਪਟ ਪ੍ਰਤੀ ਟਾਈਪੋਗ੍ਰਾਫੀ
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ਪੁਨਰ-ਰਚਿਤ
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 ਲਾਈਨ-ਉਚਾਈ
                                        css/style.css      ← ਕਾਪੀ ਕੀਤੀ
                                        js/app.js          ← ਕਾਪੀ ਕੀਤੀ
                                        images/logo.png    ← ਕਾਪੀ ਕੀਤੀ
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% ਫੌਂਟ
                                        ...
                                      de/
                                        ...
```

ਹਰ HTML ਫ਼ਾਈਲ ਵਿੱਚ `lang` ਅਤੇ `dir="rtl"` ਸ਼ਾਮਲ ਹੋ ਜਾਂਦਾ ਹੈ। ਹਰ ਸਥਾਨ ਨੂੰ ਸਹੀ ਫੌਂਟ ਸਟੈਕ, ਲਾਈਨ-ਉਚਾਈ, ਅਤੇ ਟੈਕਸਟ ਦਿਸ਼ਾ ਦੇ ਨਾਲ CSS ਮਿਲਦਾ ਹੈ। ਇੱਕ Vercel ਮਿਡਲਵੇਅਰ ਬਣਾਇਆ ਜਾਂਦਾ ਹੈ ਜੋ `Accept-Language` ਪੜ੍ਹਦਾ ਅਤੇ ਸਹੀ ਸਥਾਨ 'ਤੇ ਮੁੜ ਲਿਖਦਾ ਹੈ।

Vercel 'ਤੇ ਤੈਨਾਤ ਕਰੋ। ਟੋਕੀਓ ਵਿੱਚ ਉਪਯੋਗਕਰਤਾ Hiragino Sans ਨਾਲ 1.8 ਲਾਈਨ-ਉਚਾਈ ਵਿੱਚ ਜਪਾਨੀ ਦੇਖਦਾ ਹੈ। ਕਾਹਿਰਾ ਵਿੱਚ ਉਪਯੋਗਕਰਤਾ Noto Naskh ਨਾਲ 110% ਆਕਾਰ ਵਿੱਚ RTL ਅਰਬੀ ਦੇਖਦਾ ਹੈ। ਬੈਂਕਾਕ ਵਿੱਚ ਉਪਯੋਗਕਰਤਾ `word-break: keep-all` ਦੇ ਨਾਲ ਥਾਈ ਦੇਖਦਾ ਹੈ ਕਿਉਂਕਿ ਥਾਈ ਵਿੱਚ ਸਪੇਸ ਨਹੀਂ ਹੁੰਦੀ। ਕੋਈ ਕੌਂਫਿਗ ਨਹੀਂ।

## ਆਪਣੀਆਂ ਚਾਬੀਆਂ ਲਿਆਓ ਜਾਂ ਨਾ ਲਿਆਓ

```bash
# ਤੁਹਾਡੇ ਕੋਲ ਚਾਬੀਆਂ ਹਨ — ਸਥਾਨਕ ਤੌਰ ਚਲਦਾ ਹੈ, ਤੁਸੀਂ ਆਪਣੇ LLM ਪ੍ਰਦਾਤਾ ਨੂੰ ਸਿੱਧੇ ਭੁਗਤਾਨ ਕਰਦੇ ਹੋ
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# ਤੁਹਾਡੇ ਕੋਲ ਚਾਬੀਆਂ ਨਹੀਂ — ਬੱਸ ਕੰਮ ਕਰਦਾ ਹੈ
# ਹੋਸਟ ਕੀਤੀ ਸੇਵਾ ਆਪਣੇ ਆਪ ਵਰਤਦਾ ਹੈ
# x402 ਰਾਹੀਂ USDC ਨਾਲ ਅਨੁਵਾਦ ਪ੍ਰਤੀ ਭੁਗਤਾਨ — ਕੋਈ ਸਾਈਨਅਪ ਨਹੀਂ, ਕੋਈ ਖਾਤਾ ਨਹੀਂ
bunx translator-agent -s ./dist -l all
```

ਉਹੀ ਕਮਾਂਡ। ਜੇ API ਚਾਬੀਆਂ ਮੌਜੂਦ ਹਨ, ਇਹ ਤੁਹਾਡੇ ਪ੍ਰਦਾਤਾ ਨਾਲ ਸਥਾਨਕ ਤੌਰ ਚਲਦਾ ਹੈ। ਨਹੀਂ ਤਾਂ, ਇਹ ਹੋਸਟ ਕੀਤੀ API ਸੇ ਸੰਪਰਕ ਕਰਦਾ ਅਤੇ [x402](https://x402.org) ਰਾਹੀਂ ਪ੍ਰਤੀ ਬੇਨਤੀ ਭੁਗਤਾਨ ਕਰਦਾ — HTTP 402 ਭੁਗਤਾਨ ਪ੍ਰੋਟੋਕਾਲ। ਤੁਹਾਡਾ ਕਲਾਇੰਟ Base ਉੱਪਰ USDC ਭੁਗਤਾਨ ਕਰਦਾ, ਅਨੁਵਾਦ ਮਿਲਦੇ ਹਨ। ਕੋਈ ਪ੍ਰਮਾਣੀਕਰਨ ਨਹੀਂ, ਕੋਈ ਵਿਕਰੇਤਾ ਰਿਸ਼ਤਾ ਨਹੀਂ, ਕੋਈ ਬਿੱਲ ਨਹੀਂ।

Anthropic ਅਤੇ OpenAI ਦਾ ਸਮਰਥਨ ਕਰਦਾ ਹੈ। ਜੋ ਮਾਡਲ ਚਾਹੋ ਲਿਆਓ:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## ਕੈਸ਼ਿੰਗ

ਅਨੁਵਾਦ ਬਿਲਡ ਆਰਟੀਫੈਕਟ ਹਨ। ਬਿਲਡ ਸਮੇਂ ਬਣਾਓ, ਆਉਟਪੁੱਟ ਕੈਸ਼ ਕਰੋ, ਜਦੋਂ ਸਰੋਤ ਨਾ ਬਦਲਿਆ ਹੋਵੇ ਤਾਂ ਛੱਡ ਦਿਓ।

### Vercel

Vercel ਆਪਣੇ ਆਪ ਬਿਲਡ ਆਉਟਪੁੱਟ ਕੈਸ਼ ਕਰਦਾ ਹੈ। `postbuild` ਜੋੜੋ ਅਤੇ ਹੋ ਗਿਆ:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

ਸਰੋਤ ਨਾ ਬਦਲਿਆ = ਕੈਸ਼ ਹਿੱਟ = ਸਿਫ਼ਰ LLM ਕਾਲਾਂ = ਸਿਫ਼ਰ ਲਾਗਤ।

## ਵਿਕਲਪ

```
Usage: translator-agent [options]

Options:
  -s, --source <path>      ਸਕੈਨ ਕਰਨ ਲਈ ਸਰੋਤ ਡਾਇਰੈਕਟਰੀ ਜਾਂ ਫ਼ਾਈਲ
  -l, --locales <locales>  ਨਿਸ਼ਾਨਾ ਸਥਾਨ, ਕਾਮੇ ਨਾਲ ਅਲੱਗ ਜਾਂ 71 ਭਾਸ਼ਾਵਾਂ ਲਈ "all"
  -o, --output <path>      ਆਉਟਪੁੱਟ ਡਾਇਰੈਕਟਰੀ (ਮੂਲ: "./translations")
  -p, --provider <name>    anthropic | openai (ਮੂਲ: "anthropic")
  -m, --model <id>         ਮਾਡਲ ਓਵਰਰਾਈਡ
  -c, --concurrency <n>    ਅਧਿਕਤਮ ਸਮਾਨਾਂਤਰ LLM ਕਾਲਾਂ (ਮੂਲ: 10)
  --api-url <url>          ਹੋਸਟ ਕੀਤੀ ਸੇਵਾ URL (ਜਦੋਂ ਕੋਈ API ਚਾਬੀਆਂ ਸੈਟ ਨਹੀਂ ਤਾਂ ਆਪਣੇ-ਆਪ ਵਰਤਿਆ)
```

| ਐਕਸਟੈਨਸ਼ਨ | ਰਣਨੀਤੀ |
|---|---|
| `.json` | ਮਾਨਾਂ ਦਾ ਅਨੁਵਾਦ, ਚਾਬੀਆਂ ਸੁਰੱਖਿਤ |
| `.md` / `.mdx` | ਟੈਕਸਟ ਅਨੁਵਾਦ, ਸਿੰਟੈਕਸ ਸੁਰੱਖਿਤ |
| `.html` / `.htm` | ਟੈਕਸਟ ਅਨੁਵਾਦ, ਟੈਗ ਸੁਰੱਖਿਤ, `lang`/`dir` ਸ਼ਾਮਲ |
| ਬਾਕੀ ਸਭ | ਹਰ ਸਥਾਨ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਕਾਪੀ |

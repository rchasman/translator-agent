# translator-agent

৯০ সেকেন্ডে ১০,০০০ ডলারের লোকালাইজেশন সমস্যার সমাধান।

কোম্পানিগুলো তাদের সাইট লোকালাইজ করতে এজেন্সিকে প্রতি শব্দে $০.১০–০.২৫ দেয়। ৫,০০০ শব্দের একটি সাইট ১০টি ভাষায় করতে খরচ $৫,০০০–১২,০০০ এবং সময় লাগে ২–৪ সপ্তাহ। প্রতিবার হেডলাইন পরিবর্তন করলেই আবার সব শুরু।

এই টুল একটি কমান্ডেই এটা করে ফেলে, ৭১টি ভাষায়, আপনার বিল্ড স্টেপেই:

```bash
bunx translator-agent -s ./dist -l all
```

কোনো এজেন্সি নেই। কোনো স্প্রেডশিট নেই। কোনো ভেন্ডর লক-ইন নেই। কোনো সাইনআপ নেই। আপনার কী, আপনার বিল্ড, আপনার ভাষা।

> **আপনি এর প্রমাণই পড়ছেন।** এই README অনুবাদিত হয়েছে `bunx translator-agent -s README.md -l all` চালিয়ে। [জাপানি ভার্শনটি](./translations/ja/README.md) পড়ে দেখুন — এটা শুধু "the meter restarts" অনুবাদ করেনি, বরং জাপানি ব্যবসায়িক প্রবাদ দিয়ে বদলে দিয়েছে। [জার্মান ভার্শনটি](./translations/de/README.md) ৩০% বড় কারণ জার্মান ভাষা সবসময়ই এমন। [আরবি ভার্শনটি](./translations/ar/README.md) ডান থেকে বামে পড়া হয়। [ব্রাজিলিয়ান পর্তুগিজ ভার্শনটি](./translations/pt-BR/README.md) মনে হয় একজন ব্রাজিলিয়ানই লিখেছে, কারণ এটাই মূল উদ্দেশ্য।

## এটা কেন কাজ করে

অনুবাদ একটি সমাধানকৃত সমস্যা। লোকালাইজেশন নয়।

Google Translate "Our hamsters are working on it" কে জাপানিতে বদলে দিতে পারে। যেটা পারে না তা হলো বোঝা যে জোকটা জাপানে চলবে না, এবং এর পরিবর্তে এমন কিছু দেওয়া যা চলে — যেমন ইঞ্জিনিয়ারিং টিমের সারারাত কাজ করার প্রসঙ্গ, যা সাংস্কৃতিকভাবে উপযুক্ত এবং প্রসঙ্গে মজারও।

এই টুল অনুবাদ করে না। এটা **ট্রান্সক্রিয়েট** করে — একই প্রক্রিয়া যার জন্য অ্যাড এজেন্সিরা বিভিন্ন বাজারে ক্যাম্পেইন মানিয়ে নিতে $৫০,০০০ নেয়। তবে LLM ইতিমধ্যেই সব সংস্কৃতি, সব প্রবাদ, সব ফরম্যাটিং নিয়ম জানে। এটা জানে যে:

- `$49/month` জাপানে হয় `月額6,980円` — "$49" এর সাথে ইয়েন চিহ্ন লাগিয়ে নয়
- ব্যঙ্গ ইংরেজিতে দুর্দান্ত কিন্তু আনুষ্ঠানিক জাপানিতে মারাত্মক
- "Drowning in paperwork" ফরাসিতে হয় "noyade administrative" — একটি সত্যিকারের ফরাসি অভিব্যক্তি, শব্দে শব্দে অনুবাদ নয়
- জার্মানরা হ্যামস্টারের জোকটা রেখে দেয় কারণ Hamsterrad (হ্যামস্টারের চাকা) একটি সত্যিকারের জার্মান প্রবাদ
- ব্রাজিলিয়ানদের সহজ ভাষা দরকার নয়তো রোবট লেখা মনে হয়

মডেল প্রতিটি স্ট্রিং শ্রেণীবদ্ধ করে। UI লেবেল পায় সরাসরি অনুবাদ। মার্কেটিং কপি পায় মানিয়ে নেওয়া। হাস্যরস পায় টার্গেট কালচারের জন্য সম্পূর্ণ পুনর্নির্মাণ।

## এটা চালালে কি হয়

আপনার বিল্ড আউটপুটের দিকে পয়েন্ট করুন। এটা প্রতিটি লোকেলের জন্য পুরো ফাইল ট্রি কপি করে — টেক্সট ফাইল অনুবাদ করে, স্ট্যাটিক অ্যাসেট কপি করে, এবং ডিপ্লয়মেন্টের জন্য প্রয়োজনীয় সব কিছু তৈরি করে:

```
your-site/                          translations/
  index.html                          middleware.ts        ← লোকেল সনাক্তকরণ
  about.html             →            _locales.css         ← স্ক্রিপ্ট অনুযায়ী টাইপোগ্রাফি
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ট্রান্সক্রিয়েট
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, ১.৮ line-height
                                        css/style.css      ← কপি করা
                                        js/app.js          ← কপি করা
                                        images/logo.png    ← কপি করা
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, ১১০% ফন্ট
                                        ...
                                      de/
                                        ...
```

প্রতিটি HTML ফাইলে `lang` এবং `dir="rtl"` ইনজেক্ট হয়। প্রতিটি লোকেল পায় সঠিক ফন্ট স্ট্যাক, লাইন-হাইট, এবং টেক্সট দিক সহ CSS। একটি Vercel মিডলওয়্যার তৈরি হয় যা `Accept-Language` পড়ে এবং সঠিক লোকেলে রিরাইট করে।

Vercel এ ডিপ্লয় করুন। টোকিওর একজন ইউজার দেখবে জাপানি ভাষা Hiragino Sans এর সাথে ১.৮ লাইন-হাইটে। কায়রোর একজন ইউজার দেখবে RTL আরবি Noto Naskh এর সাথে ১১০% সাইজে। ব্যাংককের একজন ইউজার দেখবে থাই ভাষা `word-break: keep-all` সহ কারণ থাইতে ফাঁকা স্থান নেই। কোনো কনফিগ দরকার নেই।

## ৯০ সেকেন্ড, ৪ সপ্তাহ নয়

```bash
# তিনটি ভাষা, একটি JSON ফাইল
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
সম্পন্ন। ৯.৫ সেকেন্ডে ৩টি ফাইল লেখা হয়েছে

# আপনার পুরো সাইট, পৃথিবীর সব ভাষায়
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
সম্পন্ন। ৯৪ সেকেন্ডে ১৪২টি ফাইল অনুবাদ, ২৮৪টি স্ট্যাটিক ফাইল কপি
```

### আপনার বিল্ড পাইপলাইনে

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

প্রতিটি ডিপ্লয় ৭১টি ভাষায় চালু হয়। অনুবাদ হলো বিল্ড আর্টিফ্যাক্ট — ক্যাশ করা হয়, শুধু সোর্স পরিবর্তন হলেই পুনর্নির্মাণ।

## নিজের কী আনুন অথবা আনবেন না

```bash
# আপনার কাছে কী আছে — লোকালভাবে চলে, আপনি সরাসরি LLM প্রোভাইডারকে দেন
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# আপনার কাছে কী নেই — শুধু কাজ করে
# স্বয়ংক্রিয়ভাবে হোস্ট করা সার্ভিস ব্যবহার করে
# x402 এর মাধ্যমে USDC দিয়ে প্রতি অনুবাদের জন্য দিন — কোনো সাইনআপ, কোনো অ্যাকাউন্ট নেই
bunx translator-agent -s ./dist -l all
```

একই কমান্ড। API কী থাকলে এটা আপনার প্রোভাইডারের সাথে লোকালভাবে চলে। না থাকলে হোস্ট করা API হিট করে এবং [x402](https://x402.org) এর মাধ্যমে প্রতি রিকুয়েস্টে দেয় — HTTP 402 পেমেন্ট প্রোটোকল। আপনার ক্লায়েন্ট Base এ USDC দেয়, অনুবাদ পেয়ে যায়। কোনো অথ, কোনো ভেন্ডর সম্পর্ক, কোনো ইনভয়েস নেই।

Anthropic এবং OpenAI সাপোর্ট করে। যে মডেল চান সেটা নিয়ে আসুন:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## প্রতিটি স্ক্রিপ্ট সিস্টেম, হ্যান্ডেল করা

টুলটি শুধু টেক্সট অনুবাদ করে না — এটা জানে প্রতিটি লেখার পদ্ধতি কীভাবে রেন্ডার হয়:

| স্ক্রিপ্ট | কী পরিবর্তন হয় | কেন |
|---|---|---|
| **আরবি, হিব্রু, ফার্সি, উর্দু** | `dir="rtl"`, RTL ফন্ট, ১১০% সাইজ | আরবি পড়ার জন্য বড় টাইপ দরকার; পুরো লেআউট উল্টে যায় |
| **জাপানি, চীনা, কোরিয়ান** | CJK ফন্ট স্ট্যাক, ১.৮ লাইন-হাইট | অক্ষর ফিক্সড-উইডথ বর্গাকার; উল্লম্ব শ্বাসের জায়গা দরকার |
| **হিন্দি, বাংলা, তামিল, তেলুগু** | ইন্ডিক ফন্ট, ১.৮ লাইন-হাইট | মাত্রা (শিরোরেখা) অতিরিক্ত উল্লম্ব স্থান দরকার |
| **থাই** | `word-break: keep-all` | শব্দের মাঝে ফাঁক নেই — ব্রাউজারের লাইন-ব্রেক নিয়মের প্রয়োজন |
| **বার্মিজ** | ২.২ লাইন-হাইট | সব প্রধান স্ক্রিপ্টের মধ্যে সবচেয়ে লম্বা গ্লিফ |
| **খেমার** | ২.০ লাইন-হাইট | সাবস্ক্রিপ্ট ব্যঞ্জন ক্লাস্টার উল্লম্বভাবে সাজানো |

প্রতি-লোকেল CSS তৈরি:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## ক্যাশিং

অনুবাদ হলো বিল্ড আর্টিফ্যাক্ট। বিল্ড টাইমে তৈরি করুন, আউটপুট ক্যাশ করুন, সোর্স পরিবর্তন না হলে বাদ দিন।

### Vercel

Vercel স্বয়ংক্রিয়ভাবে বিল্ড আউটপুট ক্যাশ করে। `postbuild` যোগ করলেই হয়ে গেল:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

সোর্স অপরিবর্তিত = ক্যাশ হিট = শূন্য LLM কল = শূন্য খরচ।

## অপশন

```
ব্যবহার: translator-agent [অপশন]

অপশন:
  -s, --source <path>      স্ক্যান করার সোর্স ডিরেক্টরি বা ফাইল
  -l, --locales <locales>  টার্গেট লোকেল, কমা দিয়ে আলাদা বা ৭১টি ভাষার জন্য "all"
  -o, --output <path>      আউটপুট ডিরেক্টরি (ডিফল্ট: "./translations")
  -p, --provider <name>    anthropic | openai (ডিফল্ট: "anthropic")
  -m, --model <id>         মডেল ওভাররাইড
  -c, --concurrency <n>    সর্বোচ্চ সমান্তরাল LLM কল (ডিফল্ট: ১০)
  --api-url <url>          হোস্ট করা সার্ভিস URL (API কী সেট না থাকলে স্বয়ং-ব্যবহৃত)
```

| এক্সটেনশন | কৌশল |
|---|---|
| `.json` | মান অনুবাদ, কী সংরক্ষণ |
| `.md` / `.mdx` | টেক্সট অনুবাদ, সিনট্যাক্স সংরক্ষণ |
| `.html` / `.htm` | টেক্সট অনুবাদ, ট্যাগ সংরক্ষণ, `lang`/`dir` ইনজেক্ট |
| বাকি সব | প্রতিটি লোকেল ডিরেক্টরিতে কপি |

### সব ৭১টি লোকেল

ಕಂಪೆನಿಗಳು ತಮ್ಮ ವೆಬ್‌ಸೈಟ್‌ಗಳನ್ನು ಸ್ಥಳೀಕರಿಸಲು ಏಜೆನ್ಸಿಗಳಿಗೆ ಪ್ರತಿ ಪದಕ್ಕೆ ₹8–₹20 ಪಾವತಿಸುತ್ತವೆ. 5,000 ಪದಗಳ ವೆಬ್‌ಸೈಟ್ ಅನ್ನು 10 ಭಾಷೆಗಳಲ್ಲಿ ಅನುವಾದಿಸಲು ₹4,00,000–₹10,00,000 ವೆಚ್ಚವಾಗುತ್ತದೆ ಮತ್ತು 2–4 ವಾರಗಳು ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ಪ್ರತಿಬಾರಿ ನೀವು ಶೀರ್ಷಿಕೆ ಬದಲಾಯಿಸಿದಾಗ, ಮೀಟರ್ ಮತ್ತೆ ಶುರುವಾಗುತ್ತದೆ.

## ಇದು ಏಕೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ

ಅನುವಾದ ಪರಿಹರಿಸಲಾಗಿದೆ. ಸ್ಥಳೀಕರಣ ಪರಿಹರಿಸಲಾಗಿಲ್ಲ.

Google Translate "Our hamsters are working on it" ಅನ್ನು ಜಪಾನೀಸ್‌ಗೆ ಬದಲಾಯಿಸಬಹುದು. ಅದು ಮಾಡಲು ಸಾಧ್ಯವಾಗದ್ದು ಜೋಕ್ ಜಪಾನ್‌ನಲ್ಲಿ ಹಿಟ್ ಆಗುವುದಿಲ್ಲ ಎಂದು ಗುರುತಿಸಿ, ಅದರ ಬದಲಿಗೆ ಯಾವುದಾದರೂ ಹಿಟ್ ಆಗುವಂತಹ ಕಲ್ಪನೆಯೊಂದಿಗೆ ಬದಲಾಯಿಸುವುದು — ಇಂಜಿನಿಯರಿಂಗ್ ತಂಡ ರಾತ್ರಿಯೆಲ್ಲಾ ಕೆಲಸ ಮಾಡುವ ಉಲ್ಲೇಖ, ಇದು ಸಾಂಸ್ಕೃತಿಕವಾಗಿ ಸೂಕ್ತ ಮತ್ತು ಸಂದರ್ಭದಲ್ಲಿ ಹಾಸ್ಯಮಯ.

ಈ ಉಪಕರಣವು ಅನುವಾದಿಸುವುದಿಲ್ಲ. ಇದು **ಟ್ರಾನ್ಸ್‌ಕ್ರಿಯೇಟ್** ಮಾಡುತ್ತದೆ — ಅದೇ ಪ್ರಕ್ರಿಯೆ ಎಡ್ ಏಜೆನ್ಸಿಗಳು ಮಾರುಕಟ್ಟೆಗಳಲ್ಲಿ ಪ್ರಚಾರವನ್ನು ಅಳವಡಿಸುವಾಗ ₹40,00,000 ಚಾರ್ಜ್ ಮಾಡುತ್ತವೆ. ಆದರೆ LLM ಈಗಾಗಲೇ ಪ್ರತಿ ಸಂಸ್ಕೃತಿ, ಪ್ರತಿ ಗಾದೆ, ಪ್ರತಿ ಫಾರ್ಮ್ಯಾಟಿಂಗ್ ಸಂಪ್ರದಾಯವನ್ನು ತಿಳಿದಿದೆ. ಇದು ಇದನ್ನು ತಿಳಿದಿದೆ:

- `$49/month` ಜಪಾನ್‌ನಲ್ಲಿ `月額6,980円` ಆಗುತ್ತದೆ — ಯೆನ್ ಸಂಕೇತದೊಂದಿಗೆ ಸರಳವಾಗಿ "$49" ಅಲ್ಲ
- ವ್ಯಂಗ್ಯ ಇಂಗ್ಲಿಷ್‌ನಲ್ಲಿ ಯಶಸ್ವಿಯಾಗುತ್ತದೆ ಮತ್ತು ಔಪಚಾರಿಕ ಜಪಾನೀಸ್‌ನಲ್ಲಿ ವಿಫಲಗೊಳ್ಳುತ್ತದೆ
- "Drowning in paperwork" ಫ್ರೆಂಚ್‌ನಲ್ಲಿ "noyade administrative" ಆಗುತ್ತದೆ — ಒಂದು ನಿಜವಾದ ಫ್ರೆಂಚ್ ಅಭಿವ್ಯಕ್ತಿ, ಪದ-ಯ-ಪದದ ಅನುವಾದವಲ್ಲ
- ಜರ್ಮನ್‌ನವರು ಹ್ಯಾಮ್‌ಸ್ಟರ್ ಜೋಕ್ ಅನ್ನು ಇಟ್ಟುಕೊಳ್ಳುತ್ತಾರೆ ಏಕೆಂದರೆ Hamsterrad (ಹ್ಯಾಮ್‌ಸ್ಟರ್ ಚಕ್ರ) ಒಂದು ನಿಜವಾದ ಜರ್ಮನ್ ಗಾದೆ
- ಬ್ರೆಜಿಲಿಯನ್‌ನವರಿಗೆ ಅನೌಪಚಾರಿಕ ನೋಂದಣಿ ಅಗತ್ಯವಿದೆ ಅಥವಾ ಅದು ರೋಬೋಟ್ ಬರೆದಂತೆ ತೋರುತ್ತದೆ

ಮಾಡೆಲ್ ಪ್ರತಿ ಸ್ಟ್ರಿಂಗ್ ಅನ್ನು ವರ್ಗೀಕರಿಸುತ್ತದೆ. UI ಲೇಬಲ್‌ಗಳು ನೇರ ಅನುವಾದ ಪಡೆಯುತ್ತವೆ. ಮಾರ್ಕೆಟಿಂಗ್ ಕಾಪಿ ಅಳವಡಿಸಲಾಗುತ್ತದೆ. ಹಾಸ್ಯ ಗುರಿ ಸಂಸ್ಕೃತಿಗಾಗಿ ಸಂಪೂರ್ಣವಾಗಿ ಮರುಸೃಷ್ಟಿಸಲಾಗುತ್ತದೆ.

## ನೀವು ಇದನ್ನು ರನ್ ಮಾಡಿದಾಗ ಏನಾಗುತ್ತದೆ

ನಿಮ್ಮ ಬಿಲ್ಡ್ ಔಟ್‌ಪುಟ್‌ನಲ್ಲಿ ಪಾಯಿಂಟ್ ಮಾಡಿ. ಇದು ಪ್ರತಿ ಲೊಕೇಲ್‌ಗೆ ಸಂಪೂರ್ಣ ಫೈಲ್ ಟ್ರೀಯನ್ನು ಕ್ಲೋನ್ ಮಾಡುತ್ತದೆ — ಟೆಕ್ಸ್ಟ್ ಫೈಲ್‌ಗಳನ್ನು ಅನುವಾದಿಸುತ್ತದೆ, ಸ್ಟ್ಯಾಟಿಕ್ ಅಸೆಟ್‌ಗಳನ್ನು ನಕಲಿಸುತ್ತದೆ, ಮತ್ತು ಡಿಪ್ಲಾಯ್‌ಮೆಂಟ್‌ಗೆ ಅಗತ್ಯವಿರುವ ಎಲ್ಲವನ್ನೂ ಉತ್ಪಾದಿಸುತ್ತದೆ:

```
your-site/                          translations/
  index.html                          middleware.ts        ← ಲೊಕೇಲ್ ಪತ್ತೆ
  about.html             →            _locales.css         ← ಸ್ಕ್ರಿಪ್ಟ್ ಪ್ರತಿ ಟೈಪೋಗ್ರಾಫಿ
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ಟ್ರಾನ್ಸ್‌ಕ್ರಿಯೇಟೆಡ್
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 ಲೈನ್-ಹೈಟ್
                                        css/style.css      ← ನಕಲಿಸಲಾಗಿದೆ
                                        js/app.js          ← ನಕಲಿಸಲಾಗಿದೆ
                                        images/logo.png    ← ನಕಲಿಸಲಾಗಿದೆ
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% ಫಾಂಟ್
                                        ...
                                      de/
                                        ...
```

ಪ್ರತಿ HTML ಫೈಲ್ ಗೆ `lang` ಮತ್ತು `dir="rtl"` ಇಂಜೆಕ್ಟ್ ಮಾಡಲಾಗುತ್ತದೆ. ಪ್ರತಿ ಲೊಕೇಲ್ ಗೆ ಸರಿಯಾದ ಫಾಂಟ್ ಸ್ಟಾಕ್, ಲೈನ್-ಹೈಟ್, ಮತ್ತು ಟೆಕ್ಸ್ಟ್ ದಿಕ್ಕುಗಳೊಂದಿಗೆ CSS ಪಡೆಯುತ್ತದೆ. ಒಂದು Vercel middleware ಉತ್ಪಾದಿಸಲಾಗುತ್ತದೆ ಯಾವುದು `Accept-Language` ಓದಿ ಸರಿಯಾದ ಲೊಕೇಲ್‌ಗೆ ರಿರೈಟ್ ಮಾಡುತ್ತದೆ.

Vercel ಗೆ ಡಿಪ್ಲಾಯ್ ಮಾಡಿ. ಟೋಕ್ಯೋದ ಬಳಕೆದಾರರು 1.8 ಲೈನ್-ಹೈಟ್‌ನಲ್ಲಿ Hiragino Sans ಯೊಂದಿಗೆ ಜಪಾನೀಸ್ ನೋಡುತ್ತಾರೆ. ಕೈರೋದ ಬಳಕೆದಾರರು 110% ಗಾತ್ರದಲ್ಲಿ Noto Naskh ಯೊಂದಿಗೆ RTL ಅರೇಬಿಕ್ ನೋಡುತ್ತಾರೆ. ಬ್ಯಾಂಕಾಕ್‌ನ ಬಳಕೆದಾರರು `word-break: keep-all` ಯೊಂದಿಗೆ ಥಾಯ್ ನೋಡುತ್ತಾರೆ ಏಕೆಂದರೆ ಥಾಯ್‌ನಲ್ಲಿ ಸ್ಪೇಸ್ ಇರುವುದಿಲ್ಲ. ಯಾವುದೇ ಕಾನ್ಫಿಗ್ ಇಲ್ಲ.

## ನಿಮ್ಮ ಕೀಗಳನ್ನು ತಂದುಕೊಳ್ಳಿ ಅಥವಾ ತರಬೇಡಿ

```bash
# ನಿಮ್ಮ ಬಳಿ ಕೀಗಳಿವೆ — ಸ್ಥಳೀಯವಾಗಿ ರನ್ ಮಾಡುತ್ತದೆ, ನೀವು ನಿಮ್ಮ LLM ಪ್ರೊವೈಡರ್‌ಗೆ ನೇರವಾಗಿ ಪಾವತಿಸುತ್ತೀರಿ
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# ನಿಮ್ಮ ಬಳಿ ಕೀಗಳಿಲ್ಲ — ಸರಳವಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ
# ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೋಸ್ಟೆಡ್ ಸೇವೆಯನ್ನು ಬಳಸುತ್ತದೆ
# x402 ಮೂಲಕ USDC ಯೊಂದಿಗೆ ಅನುವಾದಕ್ಕಾಗಿ ಪಾವತಿಸಿ — ಯಾವುದೇ ಸೈನಪ್, ಯಾವುದೇ ಖಾತೆ ಇಲ್ಲ
bunx translator-agent -s ./dist -l all
```

ಅದೇ ಆಜ್ಞೆ. API ಕೀಗಳು ಇದ್ದರೆ, ಇದು ನಿಮ್ಮ ಪ್ರೊವೈಡರ್‌ನೊಂದಿಗೆ ಸ್ಥಳೀಯವಾಗಿ ರನ್ ಮಾಡುತ್ತದೆ. ಇಲ್ಲದಿದ್ದರೆ, ಇದು ಹೋಸ್ಟೆಡ್ API ಅನ್ನು ಹಿಟ್ ಮಾಡುತ್ತದೆ ಮತ್ತು [x402](https://x402.org) ಮೂಲಕ ವಿನಂತಿ ಪ್ರತಿ ಪಾವತಿಸುತ್ತದೆ — HTTP 402 ಪಾವತಿ ಪ್ರೋಟೋಕಾಲ್. ನಿಮ್ಮ ಕ್ಲೈಂಟ್ Base ನಲ್ಲಿ USDC ಪಾವತಿಸುತ್ತದೆ, ಅನುವಾದಗಳನ್ನು ಹಿಂಪಡೆಯುತ್ತದೆ. ಯಾವುದೇ ದೃಢೀಕರಣ, ಯಾವುದೇ ಮಾರಾಟಗಾರ ಸಂಬಂಧ, ಯಾವುದೇ ಇನ್‌ವಾಯ್ಸ್‌ಗಳು ಇಲ್ಲ.

Anthropic ಮತ್ತು OpenAI ಬೆಂಬಲಿಸುತ್ತದೆ. ನೀವು ಬಯಸುವ ಮಾಡೆಲ್ ತಂದುಕೊಳ್ಳಿ:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## ಪ್ರತಿ ಸ್ಕ್ರಿಪ್ಟ್ ಸಿಸ್ಟಮ್, ನಿರ್ವಹಿಸಲಾಗಿದೆ

ಉಪಕರಣವು ಕೇವಲ ಪಠ್ಯವನ್ನು ಅನುವಾದಿಸುವುದಿಲ್ಲ — ಇದು ಪ್ರತಿ ಬರವಣಿಗೆ ವ್ಯವಸ್ಥೆಯು ಹೇಗೆ ರೆಂಡರ್ ಆಗುತ್ತದೆ ಎಂದು ತಿಳಿದಿದೆ:

## ಕ್ಯಾಶಿಂಗ್

ಅನುವಾದಗಳು ಬಿಲ್ಡ್ ಆರ್ಟಿಫ್ಯಾಕ್ಟ್‌ಗಳು. ಬಿಲ್ಡ್ ಸಮಯದಲ್ಲಿ ಉತ್ಪಾದಿಸಿ, ಔಟ್‌ಪುಟ್ ಕ್ಯಾಶ್ ಮಾಡಿ, ಮೂಲ ಬದಲಾಗದಿದ್ದಾಗ ಸ್ಕಿಪ್ ಮಾಡಿ.

### Vercel

Vercel ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬಿಲ್ಡ್ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಕ್ಯಾಶ್ ಮಾಡುತ್ತದೆ. `postbuild` ಸೇರಿಸಿ ಮತ್ತು ನೀವು ಮುಗಿಸಿದ್ದೀರಿ:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

ಮೂಲ ಬದಲಾಗಿಲ್ಲ = ಕ್ಯಾಶ್ ಹಿಟ್ = ಶೂನ್ಯ LLM ಕಾಲ್‌ಗಳು = ಶೂನ್ಯ ವೆಚ್ಚ.

## ಆಯ್ಕೆಗಳು

```
ಬಳಕೆ: translator-agent [ಆಯ್ಕೆಗಳು]

ಆಯ್ಕೆಗಳು:
  -s, --source <path>      ಸ್ಕ್ಯಾನ್ ಮಾಡಲು ಮೂಲ ಡೈರೆಕ್ಟರಿ ಅಥವಾ ಫೈಲ್
  -l, --locales <locales>  ಗುರಿ ಲೊಕೇಲ್‌ಗಳು, ಅಲ್ಪವಿರಾಮ-ಪ್ರತ್ಯೇಕಿತ ಅಥವಾ 71 ಭಾಷೆಗಳಿಗೆ "all"
  -o, --output <path>      ಔಟ್‌ಪುಟ್ ಡೈರೆಕ್ಟರಿ (ಡೀಫಾಲ್ಟ್: "./translations")
  -p, --provider <name>    anthropic | openai (ಡೀಫಾಲ್ಟ್: "anthropic")
  -m, --model <id>         ಮಾಡೆಲ್ ಓವರ್‌ರೈಡ್
  -c, --concurrency <n>    ಗರಿಷ್ಠ ಸಮಾನಾಂತರ LLM ಕಾಲ್‌ಗಳು (ಡೀಫಾಲ್ಟ್: 10)
  --api-url <url>          ಹೋಸ್ಟೆಡ್ ಸೇವೆ URL (API ಕೀಗಳು ಸೆಟ್ ಇಲ್ಲದಿದ್ದಾಗ ಸ್ವಯಂ-ಬಳಸಲಾಗಿದೆ)
```

| ಎಕ್ಸ್‌ಟೆನ್ಷನ್ | ತಂತ್ರ |
|---|---|
| `.json` | ಮೌಲ್ಯಗಳನ್ನು ಅನುವಾದಿಸಿ, ಕೀಗಳನ್ನು ಸಂರಕ್ಷಿಸಿ |
| `.md` / `.mdx` | ಪಠ್ಯ ಅನುವಾದಿಸಿ, ಸಿಂಟ್ಯಾಕ್ಸ್ ಸಂರಕ್ಷಿಸಿ |
| `.html` / `.htm` | ಪಠ್ಯ ಅನುವಾದಿಸಿ, ಟ್ಯಾಗ್‌ಗಳನ್ನು ಸಂರಕ್ಷಿಸಿ, `lang`/`dir` ಇಂಜೆಕ್ಟ್ |
| ಎಲ್ಲದರ ಬೇರೆ | ಪ್ರತಿ ಲೊಕೇಲ್ ಡೈರೆಕ್ಟರಿಗೆ ನಕಲಿಸಿ |

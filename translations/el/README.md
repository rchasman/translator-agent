# translator-agent

Το πρόβλημα των 10.000$ στην προσαρμογή, λύθηκε σε 90 δευτερόλεπτα.

Οι εταιρίες πληρώνουν υπηρεσίες 0,10–0,25€ ανά λέξη για να προσαρμόσουν τις ιστοσελίδες τους. Μια ιστοσελίδα 5.000 λέξεων σε 10 γλώσσες κοστίζει 5.000–12.000€ και χρειάζεται 2–4 εβδομάδες. Κάθε φορά που αλλάζετε έναν τίτλο, το μετρητή ξεκινάει από την αρχή.

Αυτό το εργαλείο το κάνει με μία εντολή, σε 71 γλώσσες, κατά τη διάρκεια της διαδικασίας κατασκευής:

```bash
bunx translator-agent -s ./dist -l all
```

Χωρίς υπηρεσίες. Χωρίς υπολογιστικά φύλλα. Χωρίς εγκλωβισμό προμηθευτή. Χωρίς εγγραφή. Τα κλειδιά σας, η κατασκευή σας, οι γλώσσες σας.

> **Αυτό που διαβάζετε είναι η απόδειξη.** Αυτό το README μεταφράστηκε εκτελώντας `bunx translator-agent -s README.md -l all`. Διαβάστε την [ιαπωνική έκδοση](./translations/ja/README.md) — δεν μετέφρασε απλώς το "ο μετρητής ξεκινάει πάλι", το αντικατέστησε με ένα ιαπωνικό επιχειρηματικό ιδίωμα. ## Γιατί λειτουργεί

Η μετάφραση είναι λυμένο πρόβλημα. Η προσαρμογή δεν είναι.

Το Google Translate μπορεί να μετατρέψει το "Οι χάμστερ μας δουλεύουν πάνω σε αυτό" στα ιαπωνικά. Αυτό που δεν μπορεί να κάνει είναι να αναγνωρίσει ότι το χιούμορ δεν "κολλάει" στην Ιαπωνία, και να το αντικαταστήσει με κάτι που πετυχαίνει — όπως αναφορά στην ομάδα μηχανικών που περνάει αϋπνη νύχτα, κάτι που είναι τόσο πολιτισμικά κατάλληλο όσο και αστείο στο συγκείμενο.

Αυτό το εργαλείο δεν μεταφράζει. **Διαδημιουργεί** — την ίδια διαδικασία που χρεώνουν 50.000€ οι διαφημιστικές εταιρίες όταν προσαρμόζουν μια καμπάνια σε διαφορετικές αγορές. Εκτός από το ότι το LLM γνωρίζει ήδη κάθε πολιτισμό, κάθε ιδίωμα, κάθε σύμβαση μορφοποίησης. Ξέρει ότι:

- `49€/μήνα` γίνεται `月額6,980円` στην Ιαπωνία — όχι "49€" με ένα σύμβολο γιεν κολλημένο
- Ο σαρκασμός σπάει στα αγγλικά και πεθαίνει στα επίσημα ιαπωνικά
- Το "Πνίγομαι στη γραφειοκρατία" γίνεται "noyade administrative" στα γαλλικά — μια πραγματική γαλλική έκφραση, όχι μια λέξη προς λέξη μετάφραση
- Οι Γερμανοί κρατάνε το χιούμορ με τον χάμστερ γιατί το Hamsterrad (τροχός χάμστερ) είναι πραγματικό γερμανικό ιδίωμα
- Οι Βραζιλιάνοι χρειάζονται τον άτυπο ύφος αλλιώς ακούγεται σαν να το έγραψε ρομπότ

Το μοντέλο κατηγοριοποιεί κάθε συμβολοσειρά. Οι ετικέτες UI παίρνουν άμεση μετάφραση. Το περιεχόμενο μάρκετινγκ προσαρμόζεται. Το χιούμορ επαναδημιουργείται πλήρως για τον πολιτισμό-στόχο.

## Τι συμβαίνει όταν το εκτελείτε

Το στοχεύετε στο αποτέλεσμα της κατασκευής σας. Κλωνοποιεί ολόκληρη την ιεραρχία αρχείων ανά τοπική ρύθμιση — μεταφράζοντας αρχεία κειμένου, αντιγράφοντας στατικά στοιχεία, και δημιουργώντας όλα όσα χρειάζονται για ανάπτυξη:

```
your-site/                          translations/
  index.html                          middleware.ts        ← ανίχνευση τοπικής ρύθμισης
  about.html             →            _locales.css         ← τυπογραφία ανά γραφή
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", διαδημιουργημένο
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 ύψος γραμμής
                                        css/style.css      ← αντιγραμμένο
                                        js/app.js          ← αντιγραμμένο
                                        images/logo.png    ← αντιγραμμένο
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% γραμματοσειρά
                                        ...
                                      de/
                                        ...
```

Κάθε αρχείο HTML παίρνει `lang` και `dir="rtl"` ενσωματωμένα. Κάθε τοπική ρύθμιση παίρνει CSS με τη σωστή στοίβα γραμματοσειρών, ύψος γραμμής, και κατεύθυνση κειμένου. Δημιουργείται ένα middleware Vercel που διαβάζει το `Accept-Language` και ανακατευθύνει στη σωστή τοπική ρύθμιση.

Αναπτύσσετε στο Vercel. Ένας χρήστης στο Τόκιο βλέπει ιαπωνικά με Hiragino Sans σε 1.8 ύψος γραμμής. Ένας χρήστης στο Κάιρο βλέπει RTL αραβικά με Noto Naskh σε μέγεθος 110%. Ένας χρήστης στη Μπανγκόκ βλέπει ταϊλανδικά με `word-break: keep-all` γιατί τα ταϊλανδικά δεν έχουν κενά. Χωρίς ρυθμίσεις.

```bash
# Τρεις γλώσσες, ένα JSON αρχείο
$ bunx translator-agent -s ./messages.json -l fr,de,ja

  [67%] de/messages.json
  [100%] ja/messages.json
Τέλος. 3 αρχεία γραμμένα σε 9.5δ

# Ολόκληρη η ιστοσελίδα σας, κάθε γλώσσα στη γη
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
Τέλος. 142 αρχεία μεταφράστηκαν, 284 στατικά αρχεία αντιγράφτηκαν σε 94δ
```

### Στη διαδικασία κατασκευής σας

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

Κάθε ανάπτυξη αποστέλλεται σε 71 γλώσσες. Οι μεταφράσεις είναι προϊόντα κατασκευής — αποθηκευμένες προσωρινά, αναδημιουργούνται μόνο όταν αλλάζει η πηγή.

## Φέρτε τα δικά σας κλειδιά ή όχι

```bash
# Έχετε κλειδιά — εκτελείται τοπικά, πληρώνετε απευθείας τον πάροχο LLM
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# Δεν έχετε κλειδιά — απλώς λειτουργεί
# Αυτόματα χρησιμοποιεί την εξυπηρετούμενη υπηρεσία
# Πληρώστε ανά μετάφραση με USDC μέσω x402 — χωρίς εγγραφή, χωρίς λογαριασμό
bunx translator-agent -s ./dist -l all
```

Ίδια εντολή. Αν υπάρχουν API κλειδιά, εκτελείται τοπικά με τον πάροχό σας. Αν όχι, απευθύνεται στο εξυπηρετούμενο API και πληρώνει ανά αίτημα μέσω [x402](https://x402.org) — το πρωτόκολλο πληρωμής HTTP 402. Ο πελάτης σας πληρώνει USDC στο Base, παίρνει πίσω μεταφράσεις. Χωρίς πιστοποίηση, χωρίς σχέση προμηθευτή, χωρίς τιμολόγια.

Υποστηρίζει Anthropic και OpenAI. Φέρτε όποιο μοντέλο θέλετε:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Κάθε σύστημα γραφής, διαχειρισμένο

Το εργαλείο δεν απλώς μεταφράζει κείμενο — γνωρίζει πώς εμφανίζεται κάθε σύστημα γραφής:

| Γραφή | Τι αλλάζει | Γιατί |
|---|---|---|
| **Αραβικά, Εβραϊκά, Φάρσι, Ουρντού** | `dir="rtl"`, γραμματοσειρές RTL, μέγεθος 110% | Τα αραβικά χρειάζονται μεγαλύτερη γραμματοσειρά για να είναι ευανάγνωστα· όλη η διάταξη αντικατοπτρίζεται |
| **Ιαπωνικά, Κινεζικά, Κορεατικά** | Στοίβες CJK γραμματοσειρών, ύψος γραμμής 1.8 | Οι χαρακτήρες είναι τετράγωνα σταθερού πλάτους· χρειάζονται κατακόρυφο χώρο αναπνοής |
| **Χίντι, Μπεγκάλι, Ταμίλ, Τελούγκου** | Ινδικές γραμματοσειρές, ύψος γραμμής 1.8 | Οι κεφαλικές γραμμές (shirorekha) χρειάζονται επιπλέον κατακόρυφο χώρο |
| **Ταϊλανδικά** | `word-break: keep-all` | Δεν υπάρχουν κενά μεταξύ λέξεων — ο περιηγητής χρειάζεται ρητούς κανόνες αλλαγής γραμμής |
| **Βιρμανέζικα** | Ύψος γραμμής 2.2 | Τα πιο ψηλά γλυφα από κάθε μεγάλη γραφή |
| **Χμερ** | Ύψος γραμμής 2.0 | Τα συμπλέγματα δείκτη συμφώνων στοιβάζονται κατακόρυφα |

Δημιουργημένο CSS ανά τοπική ρύθμιση:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Προσωρινή αποθήκευση

Οι μεταφράσεις είναι προϊόντα κατασκευής. Δημιουργούνται κατά την κατασκευή, αποθηκεύεται το αποτέλεσμα, παραλείπονται όταν η πηγή δεν έχει αλλάξει.

### Vercel

Το Vercel αποθηκεύει αυτόματα προσωρινά το αποτέλεσμα κατασκευής. Προσθέστε `postbuild` και τελειώσατε:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Πηγή αμετάβλητη = επιτυχία προσωρινής αποθήκευσης = μηδέν κλήσεις LLM = μηδέν κόστος.

## Επιλογές

```
Χρήση: translator-agent [επιλογές]

Επιλογές:
  -s, --source <path>      κατάλογος ή αρχείο πηγής για σάρωση
  -l, --locales <locales>  τοπικές ρυθμίσεις στόχου, χωρισμένες με κόμματα ή "all" για 71 γλώσσες
  -o, --output <path>      κατάλογος εξόδου (προεπιλογή: "./translations")
  -p, --provider <name>    anthropic | openai (προεπιλογή: "anthropic")
  -m, --model <id>         παράκαμψη μοντέλου
  -c, --concurrency <n>    μέγιστες παράλληλες κλήσεις LLM (προεπιλογή: 10)
  --api-url <url>          URL εξυπηρετούμενης υπηρεσίας (αυτόματη χρήση όταν δεν υπάρχουν API κλειδιά)
```

| Επέκταση | Στρατηγική |
|---|---|
| `.json` | Μετάφραση αξιών, διατήρηση κλειδιών |
| `.md` / `.mdx` | Μετάφραση κειμένου, διατήρηση σύνταξης |
| `.html` / `.htm` | Μετάφραση κειμένου, διατήρηση ετικετών, ενσωμάτωση `lang`/`dir` |
| Όλα τα υπόλοιπα | Αντιγραφή σε κάθε κατάλογο τοπικής ρύθμισης |

### Όλες οι 71 τοπικές ρυθμίσεις

Το `-l all` καλύπτει ~95% των χρηστών internet: zh-CN, zh-TW, ja, ko, vi, th, id, ms, fil, my, hi, bn, ta, te, mr, gu, kn, ml, pa, ur, fa, tr, he, ar, kk, uz, fr, de, es, pt, pt-BR, it, nl, ca, gl, sv, da, no, fi, is, pl, cs, sk, hu, ro, bg, hr, sr, sl, uk, ru, lt, lv, et, el, ga, sw, am, ha, yo, zu, af, km, lo, ne, si, ka, az, mn

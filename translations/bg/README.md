# translator-agent

Проблемът с локализацията за 10 000 долара, решен за 90 секунди.

Компаниите плащат на агенции 0,10-0,25 лева на дума за локализиране на сайтовете си. Сайт от 5000 думи на 10 езика струва 5000-12000 лева и отнема 2-4 седмици. Всеки път когато промените заглавие, таймерът започва отново.

Този инструмент го прави с една команда, на 71 езика, по време на процеса на build:

```bash
bunx translator-agent -s ./dist -l all
```

Без агенция. Без електронни таблици. Без vendor lock-in. Без регистрация. Вашите ключове, вашият build, вашите езици.

## Защо това работи

Преводът е решен проблем. Локализацията не е.

Google Translate може да превърне "Нашите хомяци работят по въпроса" на японски. Това, което не може да направи, е да разпознае, че шегата не е смешна в Япония, и да я замени с нещо друго — като споменаване на инженерния екип, който прекарва цяла нощ в работа, което е и културно подходящо, и смешно в контекста.

Този инструмент не превежда. Той **транскреира** — същия процес, за който рекламните агенции таксуват 50 000 долара при адаптиране на кампания в различни пазари. С разликата, че LLM-ът вече знае всяка култура, всеки идиом, всяка конвенция за форматиране. Той знае, че:

- `49 лв./месечно` става `月額6,980円` в Япония — не "49 лв." с йенски символ залепен отгоре
- Сарказмът убива на английски и умира в официалния японски
- "Удавям се в документи" става "noyade administrative" на френски — реален френски израз, не буквален превод
- Германците запазват шегата с хомяците, защото Hamsterrad (колело за хомяци) е истински немски идиом
- Бразилците се нуждаят от неформален регистър, иначе звучи като написано от робот

Моделът класифицира всеки string. UI етикетите получават директен превод. Маркетинговото съдържание се адаптира. Хуморът се пресъздава изцяло за целевата култура.

## Какво се случва когато го стартирате

Насочете го към вашия build output. То клонира цялото файлово дърво за всеки локал — превежда текстови файлове, копира статични ресурси и генерира всичко необходимо за deployment:

```
your-site/                          translations/
  index.html                          middleware.ts        ← разпознаване на локал
  about.html             →            _locales.css         ← типография за всяка азбука
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", транскрейрано
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← копирано
                                        js/app.js          ← копирано
                                        images/logo.png    ← копирано
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% шрифт
                                        ...
                                      de/
                                        ...
```

Всеки HTML файл получава `lang` и `dir="rtl"` атрибути. Всеки локал получава CSS с правилния font stack, line-height и посока на текста. Генерира се Vercel middleware, който чете `Accept-Language` и пренасочва към правилния локал.

Deploy към Vercel. Потребител в Токио вижда японски с Hiragino Sans на 1.8 line-height. Потребител в Кайро вижда RTL арабски с Noto Naskh на 110% размер. Потребител в Банкок вижда тайландски с `word-break: keep-all`, защото тайландският няма интервали. Без настройка.

## Донесете си собствените ключове или не

```bash
# Имате ключове — работи локално, плащате директно на LLM доставчика
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# Нямате ключове — просто работи
# Автоматично използва хостваната услуга
# Плащате за превод с USDC чрез x402 — без регистрация, без акаунт
bunx translator-agent -s ./dist -l all
```

Същата команда. Ако има API ключове, работи локално с вашия доставчик. Ако няма, удря хостваното API и плаща за заявка чрез [x402](https://x402.org) — HTTP 402 payment протоколът. Клиентът ви плаща USDC на Base, получава преводи обратно. Без автентикация, без vendor връзка, без фактури.

Поддържа Anthropic и OpenAI. Донесете който модел искате:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Всяка писмена система, обработена

Инструментът не просто превежда текст — той знае как всяка писмена система се рендерира:

| Азбука | Какво се променя | Защо |
|---|---|---|
| **Арабски, иврит, фарси, урду** | `dir="rtl"`, RTL шрифтове, 110% размер | Арабският се нуждае от по-голям шрифт за четимост; целият layout се огледалва |
| **Японски, китайски, корейски** | CJK font stacks, 1.8 line-height | Знаците са квадрати с фиксирана ширина; трябва им вертикално пространство |
| **Хинди, бенгалски, тамилски, телугу** | Indic шрифтове, 1.8 line-height | Горните черти (shirorekha) се нуждаят от допълнително вертикално място |
| **Тайландски** | `word-break: keep-all` | Няма интервали между думите — браузърът се нуждае от изрични правила за пренасяне |
| **Бирмански** | 2.2 line-height | Най-високите глифове от всички основни азбуки |
| **Кхмерски** | 2.0 line-height | Съгласните под реда се подреждат вертикално |

Генериран CSS за всеки локал:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Кеширане

Преводите са build артефакти. Генерират се по време на build, кешират се изходът, прескачат се когато източникът не се е променил.

### Vercel

Vercel кешира build output автоматично. Добавете `postbuild` и сте готови:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Източник непроменен = cache hit = нула LLM извиквания = нула разходи.

## Опции

```
Usage: translator-agent [options]

Options:
  -s, --source <path>      източна директория или файл за сканиране
  -l, --locales <locales>  целеви локали, разделени със запетая или "all" за 71 езика
  -o, --output <path>      изходна директория (по подразбиране: "./translations")
  -p, --provider <name>    anthropic | openai (по подразбиране: "anthropic")
  -m, --model <id>         замяна на модел
  -c, --concurrency <n>    максимални паралелни LLM извиквания (по подразбиране: 10)
  --api-url <url>          URL на хостваната услуга (използва се автоматично когато няма API ключове)
```

| Разширение | Стратегия |
|---|---|
| `.json` | Превеждай стойностите, запази ключовете |
| `.md` / `.mdx` | Превеждай текста, запази синтаксиса |
| `.html` / `.htm` | Превеждай текста, запази таговете, вмъкни `lang`/`dir` |
| Всичко останало | Копирай в директорията на всеки локал |

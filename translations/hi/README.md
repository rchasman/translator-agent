# translator-agent

₹8,00,000 का स्थानीयकरण समस्या, 90 सेकंड में हल।

कंपनियां अपनी साइटों को स्थानीयकृत करने के लिए एजेंसियों को ₹8–20 प्रति शब्द का भुगतान करती हैं। 5,000 शब्दों की साइट को 10 भाषाओं में रूपांतरित करने की लागत ₹4,00,000–9,60,000 होती है और 2–4 सप्ताह का समय लगता है। हर बार जब आप एक शीर्षक बदलते हैं, तो खर्चा फिर से शुरू हो जाता है।

यह उपकरण एक ही कमांड में, 71 भाषाओं में, आपके बिल्ड स्टेप के दौरान यह काम करता है:

```bash
bunx translator-agent -s ./dist -l all
```

कोई एजेंसी नहीं। कोई स्प्रेडशीट नहीं। कोई वेंडर लॉक-इन नहीं। कोई साइनअप नहीं। आपकी चाबियां, आपका बिल्ड, आपकी भाषाएं।

> **आप प्रमाण पढ़ रहे हैं।** यह README `bunx translator-agent -s README.md -l all` चलाकर अनुवादित किया गया था। जाकर [जापानी संस्करण](./translations/ja/README.md) पढ़िए — इसने सिर्फ "the meter restarts" का अनुवाद नहीं किया, बल्कि इसे जापानी व्यावसायिक मुहावरे से बदल दिया। [जर्मन संस्करण](./translations/de/README.md) 30% लंबा है क्योंकि जर्मन हमेशा होता है। [अरबी संस्करण](./translations/ar/README.md) दाएं-से-बाएं पढ़ा जाता है। [ब्राजीलियाई पुर्तगाली संस्करण](./translations/pt-BR/README.md) ऐसे लगता है जैसे किसी ब्राजीलियाई ने लिखा हो, क्योंकि यही मुद्दा है।

## यह क्यों काम करता है

अनुवाद एक हल की गई समस्या है। स्थानीयकरण नहीं है।

Google Translate "हमारे हम्सटर इस पर काम कर रहे हैं" को जापानी में बदल सकता है। जो यह नहीं कर सकता वह यह पहचानना है कि यह मजाक जापान में नहीं जमता, और इसे ऐसी चीज़ से बदलना है जो जमे — जैसे इंजीनियरिंग टीम के पूरी रात जगकर काम करने का संदर्भ देना, जो सांस्कृतिक रूप से उपयुक्त और संदर्भ में मज़ेदार दोनों है।

यह उपकरण अनुवाद नहीं करता। यह **ट्रांसक्रिएट** करता है — वही प्रक्रिया जिसके लिए विज्ञापन एजेंसियां मार्केट्स में अभियान अनुकूलित करते समय ₹40,00,000 चार्ज करती हैं। सिवाय इसके कि LLM पहले से ही हर संस्कृति, हर मुहावरे, हर स्वरूपण परंपरा को जानता है। यह जानता है कि:

- `$49/month` जापान में `月額6,980円` बन जाता है — येन सिंबल लगे "$49" नहीं
- व्यंग्य अंग्रेजी में घातक है और औपचारिक जापानी में मर जाता है
- "कागजी कार्रवाई में डूबना" फ्रेंच में "noyade administrative" बन जाता है — एक असली फ्रेंच अभिव्यक्ति, शब्द-दर-शब्द अनुवाद नहीं
- जर्मन हम्सटर जोक रखते हैं क्योंकि Hamsterrad (हम्सटर व्हील) एक असली जर्मन मुहावरा है
- ब्राजीलियाई को अनौपचारिक स्वर चाहिए वरना यह ऐसे लगता है जैसे रोबोट ने लिखा हो

मॉडल हर स्ट्रिंग को वर्गीकृत करता है। UI लेबल्स को सीधा अनुवाद मिलता है। मार्केटिंग कॉपी को अनुकूलित किया जाता है। हास्य को लक्षित संस्कृति के लिए पूरी तरह से फिर से बनाया जाता है।

## जब आप इसे चलाते हैं तो क्या होता है

इसे अपने बिल्ड आउटपुट पर पॉइंट करें। यह प्रति लोकेल पूरे फाइल ट्री को क्लोन करता है — टेक्स्ट फाइलों का अनुवाद करता है, स्टेटिक एसेट्स कॉपी करता है, और तैनाती के लिए आवश्यक सब कुछ जेनरेट करता है:

```
your-site/                          translations/
  index.html                          middleware.ts        ← लोकेल डिटेक्शन
  about.html             →            _locales.css         ← स्क्रिप्ट के अनुसार टाइपोग्राफी
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ट्रांसक्रिएटेड
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 लाइन-हाइट
                                        css/style.css      ← कॉपी किया गया
                                        js/app.js          ← कॉपी किया गया
                                        images/logo.png    ← कॉपी किया गया
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% फॉन्ट
                                        ...
                                      de/
                                        ...
```

हर HTML फाइल में `lang` और `dir="rtl"` इंजेक्ट हो जाता है। हर लोकेल को सही फॉन्ट स्टैक, लाइन-हाइट, और टेक्स्ट दिशा के साथ CSS मिलता है। एक Vercel मिडलवेयर जेनरेट होता है जो `Accept-Language` पढ़ता है और सही लोकेल पर रीराइट करता है।

Vercel पर तैनात करें। टोक्यो में एक उपयोगकर्ता को 1.8 लाइन-हाइट पर Hiragino Sans के साथ जापानी दिखेगा। काहिरा में एक उपयोगकर्ता को 110% आकार पर Noto Naskh के साथ RTL अरबी दिखेगा। बैंकॉक में एक उपयोगकर्ता को `word-break: keep-all` के साथ थाई दिखेगा क्योंकि थाई में स्पेस नहीं होते। कोई कॉन्फ़िग नहीं।

## 90 सेकंड, 4 सप्ताह नहीं

```bash
# तीन भाषाएं, एक JSON फाइल
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
हो गया। 9.5s में 3 फाइलें लिखी गईं

# आपकी पूरी साइट, धरती की हर भाषा
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
हो गया। 142 फाइलें अनुवादित, 284 स्टेटिक फाइलें 94s में कॉपी की गईं
```

## अपनी चाबियां लाएं या न लाएं

```bash
# आपके पास चाबियां हैं — लोकल रन होता है, आप अपने LLM प्रदाता को सीधे भुगतान करते हैं
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# आपके पास चाबियां नहीं हैं — बस काम करता है
# होस्टेड सेवा का स्वचालित उपयोग
# x402 के माध्यम से USDC के साथ प्रति अनुवाद भुगतान — कोई साइनअप नहीं, कोई खाता नहीं
bunx translator-agent -s ./dist -l all
```

वही कमांड। अगर API चाबियां मौजूद हैं, तो यह आपके प्रदाता के साथ लोकल चलता है। अगर नहीं, तो यह होस्टेड API को हिट करता है और [x402](https://x402.org) के माध्यम से प्रति अनुरोध भुगतान करता है — HTTP 402 भुगतान प्रोटोकॉल। आपका क्लाइंट Base पर USDC का भुगतान करता है, अनुवाद वापस मिलते हैं। कोई ऑथ नहीं, कोई वेंडर रिश्ता नहीं, कोई चालान नहीं।

Anthropic और OpenAI को सपोर्ट करता है। जो भी मॉडल चाहते हैं लाएं:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## हर स्क्रिप्ट सिस्टम, संभाला गया

यह उपकरण सिर्फ टेक्स्ट का अनुवाद नहीं करता — यह जानता है कि हर लेखन प्रणाली कैसे रेंडर होती है:

| स्क्रिप्ट | क्या बदलता है | क्यों |
|---|---|---|
| **अरबी, हिब्रू, फारसी, उर्दू** | `dir="rtl"`, RTL फॉन्ट्स, 110% आकार | अरबी को पढ़ने योग्य होने के लिए बड़े टाइप की जरूरत; पूरा लेआउट मिरर होता है |
| **जापानी, चीनी, कोरियाई** | CJK फॉन्ट स्टैक्स, 1.8 लाइन-हाइट | अक्षर फिक्स्ड-विड्थ वर्ग हैं; वर्टिकल सांस लेने की जगह चाहिए |
| **हिंदी, बंगाली, तमिल, तेलुगू** | इंडिक फॉन्ट्स, 1.8 लाइन-हाइट | हेडस्ट्रोक्स (शिरोरेखा) को अतिरिक्त वर्टिकल स्पेस चाहिए |
| **थाई** | `word-break: keep-all` | शब्दों के बीच कोई स्पेस नहीं — ब्राउज़र को स्पष्ट लाइन-ब्रेक नियम चाहिए |
| **बर्मीज़** | 2.2 लाइन-हाइट | किसी भी मुख्य स्क्रिप्ट के सबसे लंबे ग्लिफ्स |
| **खमेर** | 2.0 लाइन-हाइट | सबस्क्रिप्ट व्यंजन क्लस्टर वर्टिकली स्टैक होते हैं |

जेनरेटेड प्रति-लोकेल CSS:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## कैशिंग

अनुवाद बिल्ड आर्टिफैक्ट्स हैं। बिल्ड टाइम पर जेनरेट करें, आउटपुट कैश करें, जब स्रोत नहीं बदला हो तो स्किप करें।

### Vercel

Vercel बिल्ड आउटपुट को स्वचालित रूप से कैश करता है। `postbuild` जोड़ें और आप तैयार हैं:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

स्रोत अपरिवर्तित = कैश हिट = शून्य LLM कॉल्स = शून्य लागत।

## विकल्प

```
उपयोग: translator-agent [options]

विकल्प:
  -s, --source <path>      स्कैन करने के लिए स्रोत डायरेक्टरी या फाइल
  -l, --locales <locales>  लक्षित लोकेल्स, कॉमा-सेपरेटेड या 71 भाषाओं के लिए "all"
  -o, --output <path>      आउटपुट डायरेक्टरी (डिफ़ॉल्ट: "./translations")
  -p, --provider <name>    anthropic | openai (डिफ़ॉल्ट: "anthropic")
  -m, --model <id>         मॉडल ओवरराइड
  -c, --concurrency <n>    मैक्स पैरेलल LLM कॉल्स (डिफ़ॉल्ट: 10)
  --api-url <url>          होस्टेड सेवा URL (जब कोई API चाबियां सेट नहीं हों तो ऑटो-उपयोग)
```

| एक्सटेंशन | रणनीति |
|---|---|
| `.json` | मानों का अनुवाद, चाबियां संरक्षित |
| `.md` / `.mdx` | टेक्स्ट अनुवाद, सिंटैक्स संरक्षित |
| `.html` / `.htm` | टेक्स्ट अनुवाद, टैग संरक्षित, `lang`/`dir` इंजेक्ट |
| बाकी सब | हर लोकेल डायरेक्टरी में कॉपी |

### सभी 71 लोकेल्स

`-l all` ~95% इंटरनेट उपयोगकर्ताओं को कवर करता है: zh-CN, zh-TW, ja, ko, vi, th, id, ms, fil, my, hi, bn, ta, te, mr, gu, kn, ml, pa, ur, fa, tr, he, ar, kk, uz, fr, de, es, pt, pt-BR, it, nl, ca, gl, sv, da, no, fi, is, pl, cs, sk, hu, ro, bg, hr, sr, sl, uk, ru, lt, lv, et, el, ga, sw, am, ha, yo, zu, af, km, lo, ne, si, ka, az, mn

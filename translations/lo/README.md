# translator-agent

ໂຄງການການແປພາສາ $10,000 ແກ້ໄຂໃນ 90 ວິນາທີ

ບໍລິສັດຕ່າງໆຈ່າຍໃຫ້ບໍລິສັດຮັບເໝົາ $0.10–0.25 ຕໍ່ຄໍາເພື່ອປັບປຸງເວັບໄຊໃຫ້ເໝາະກັບທ້ອງຖິ່ນ. ເວັບໄຊ 5,000 ຄໍາໃສ່ 10 ພາສາໃຊ້ຄ່າໃຊ້ຈ່າຍ $5,000–12,000 ແລະໃຊ້ເວລາ 2–4 ອາທິດ. ທຸກຄັ້ງທີ່ທ່ານປ່ຽນຫົວຂໍ້, ການນັບເວລາເລີ່ມຕົ້ນໃໝ່.

ເຄື່ອງມືນີ້ເຮັດໃນຄໍາສັ່ງດຽວ, ໃສ່ 71 ພາສາ, ໃນຂະບວນການສ້າງຂອງທ່ານ:

```bash
bunx translator-agent -s ./dist -l all
```

ບໍ່ມີບໍລິສັດຮັບເໝົາ. ບໍ່ມີສະປຣັດຊີດ. ບໍ່ມີການຜູກມັດກັບຜູ້ສະໜອງ. ບໍ່ຕ້ອງສະໝັກສະມາຊິກ. ກະແຈຂອງທ່ານ, ການສ້າງຂອງທ່ານ, ພາສາຂອງທ່ານ.

## ເປັນຫຍັງນີ້ຈຶ່ງເຮັດວຽກ

ການແປພາສາແມ່ນບັນຫາທີ່ແກ້ໄດ້ແລ້ວ. ການປັບປຸງໃຫ້ເໝາະກັບທ້ອງຖິ່ນຍັງບໍ່ທັນ.

Google Translate ສາມາດເປີ້ນ "ໜູນ້ອຍຂອງພວກເຮົາກໍາລັງເຮັດວຽກຢູ່" ໃສ່ພາສາຍີ່ປຸ່ນໄດ້. ສິ່ງທີ່ມັນເຮັດບໍ່ໄດ້ແມ່ນຮັບຮູ້ວ່າເລື່ອງຕະຫລົກນັ້ນບໍ່ເຂົ້າກັບຍີ່ປຸ່ນ, ແລະແທນທີ່ດ້ວຍສິ່ງທີ່ເຂົ້າກັນໄດ້ — ເຊັ່ນການອ້າງອີງເຖິງທີມວິສະວະກໍາທີ່ເຮັດວຽກຂ້າມຄືນ, ເຊິ່ງທັງເໝາະສົມກັບວັດທະນະທໍາແລະຕະຫລົກໃນບໍລິບົດນັ້ນ.

ເຄື່ອງມືນີ້ບໍ່ໄດ້ແປ. ມັນ **ສ້າງຄືນ** — ຂະບວນການດຽວກັບທີ່ບໍລິສັດໂຄສະນາຄິດຄ່າ $50,000 ສໍາລັບການປັບແຕ່ງແຄມເປນຕ່າງໆໃນຕະຫຼາດຕ່າງໆ. ນອກຈາກ LLM ຮູ້ຈັກທຸກວັດທະນະທໍາ, ທຸກສໍານວນ, ທຸກແບບຟໍມແລ້ວ. ມັນຮູ້ວ່າ:

- `$49/month` ກາຍເປັນ `月額6,980円` ໃນຍີ່ປຸ່ນ — ບໍ່ແມ່ນ "$49" ກັບສັນຍາລັກເຢນແປະໄວ້
- ການເຍາະເຍີ້ຍເຮັດວຽກໄດ້ດີໃນພາສາອັງກິດແລະຕາຍໃນພາສາຍີ່ປຸ່ນຢ່າງເປັນທາງການ
- "ຈົມໃນເອກະສານ" ກາຍເປັນ "noyade administrative" ໃນພາສາຝຣັ່ງ — ສໍານວນຝຣັ່ງທີ່ແທ້ຈິງ, ບໍ່ແມ່ນການແປຄໍາຕໍ່ຄໍາ
- ຄົນເຢຍລະມັນຮັກສາເລື່ອງຕະຫລົກນີ້ເພາະ Hamsterrad (ລໍ້ໜູນ້ອຍ) ແມ່ນສໍານວນເຢຍລະມັນທີ່ແທ້ຈິງ
- ຄົນບຣາຊິວຕ້ອງການລະດັບການສື່ສານແບບໃກ້ຊິດຖ້າບໍ່ດັ່ງນັ້ນຈະຟັງເໝືອນຫຸ່ນຍົນຂຽນ

ໂມເດວຈັດປະເພດແຕ່ລະສະຕຣິງ. ປ້າຍຫນ້າຕ່າງຜູ້ໃຊ້ໄດ້ຮັບການແປໂດຍກົງ. ເນື້ອຫາການຕະຫຼາດໄດ້ຮັບການປັບປຸງ. ຄວາມຕະຫລົກໄດ້ຮັບການສ້າງຄືນຢ່າງສົມບູນສໍາລັບວັດທະນະທໍາເປົ້າໝາຍ.

## ເກີດຫຍັງຂຶ້ນເມື່ອທ່ານເອີ້ນໃຊ້

ຊີ້ໄປທີ່ຜົນໄດ້ຮັບຈາກການສ້າງຂອງທ່ານ. ມັນກ໊ອບປີໂຄງສ້າງໄຟລ์ທັງໝົດຕໍ່ທ້ອງຖິ່ນ — ການແປໄຟລ์ຂໍ້ຄວາມ, ການກ໊ອບປີຊັບສິນຄົງທີ່, ແລະການສ້າງທຸກສິ່ງທີ່ຈໍາເປັນສໍາລັບການນໍາໄປໃຊ້:

```
your-site/                          translations/
  index.html                          middleware.ts        ← ການກວດພົບທ້ອງຖິ່ນ
  about.html             →            _locales.css         ← ການພິມພີຕາມຕົວໜັງສື
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", ສ້າງຄືນ
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, ຄວາມສູງແຖວ 1.8
                                        css/style.css      ← ກ໊ອບປີ
                                        js/app.js          ← ກ໊ອບປີ
                                        images/logo.png    ← ກ໊ອບປີ
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, ຟ໋ອນ 110%
                                        ...
                                      de/
                                        ...
```

ທຸກໄຟລ์ HTML ໄດ້ຮັບ `lang` ແລະ `dir="rtl"` ໄດ້ຮັບການແຊກ. ທຸກທ້ອງຖິ່ນໄດ້ຮັບ CSS ພ້ອມກັບຊຸດຟ໋ອນທີ່ຖືກຕ້ອງ, ຄວາມສູງແຖວ, ແລະທິດທາງຂໍ້ຄວາມ. Middleware Vercel ຖືກສ້າງທີ່ອ່ານ `Accept-Language` ແລະຂຽນຄືນໄປທີ່ທ້ອງຖິ່ນທີ່ຖືກຕ້ອງ.

ນໍາໄປໃຊ້ໃນ Vercel. ຜູ້ໃຊ້ໃນໂຕກຽວເບິ່ງພາສາຍີ່ປຸ່ນດ້ວຍ Hiragino Sans ທີ່ຄວາມສູງແຖວ 1.8. ຜູ້ໃຊ້ໃນໄຄໂຣເບິ່ງອາຣັບ RTL ດ້ວຍ Noto Naskh ທີ່ຂະໜາດ 110%. ຜູ້ໃຊ້ໃນກຸງເທບເບິ່ງພາສາໄທດ້ວຍ `word-break: keep-all` ເພາະພາສາໄທບໍ່ມີຊ່ອງຫວ່າງ. ບໍ່ມີການຕັ້ງຄ່າ.

## ນໍາກະແຈຂອງທ່ານເອງມາ ຫຼືບໍ່ກໍ່ເອົາ

```bash
# ທ່ານມີກະແຈ — ເຮັດວຽກທ້ອງຖິ່ນ, ທ່ານຈ່າຍໃຫ້ຜູ້ສະໜອງ LLM ໂດຍກົງ
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# ທ່ານບໍ່ມີກະແຈ — ເຮັດວຽກໄດ້ເລີຍ
# ໃຊ້ບໍລິການ hosted ໂດຍອັດຕະໂນມັດ
# ຈ່າຍຕໍ່ການແປດ້ວຍ USDC ຜ່ານ x402 — ບໍ່ຕ້ອງສະໝັກ, ບໍ່ມີບັນຊີ
bunx translator-agent -s ./dist -l all
```

ຄໍາສັ່ງດຽວກັນ. ຖ້າມີກະແຈ API, ມັນເຮັດວຽກທ້ອງຖິ່ນກັບຜູ້ສະໜອງຂອງທ່ານ. ຖ້າບໍ່, ມັນໄປທີ່ API hosted ແລະຈ່າຍຕໍ່ຄໍາຮ້ອງຂໍຜ່ານ [x402](https://x402.org) — ໂປຣໂຕຄອລການຊໍາລະ HTTP 402. ລູກຄ້າຂອງທ່ານຈ່າຍ USDC ໃນ Base, ໄດ້ຮັບການແປຄືນ. ບໍ່ມີການພິສູດຕົວຕົນ, ບໍ່ມີຄວາມສໍາພັນກັບຜູ້ສະໜອງ, ບໍ່ມີໃບແຈ້ງໜີ້.

ສະໜັບສະໜູນ Anthropic ແລະ OpenAI. ນໍາໂມເດວທີ່ທ່ານຕ້ອງການມາ:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## ທຸກລະບົບຕົວໜັງສື, ຈັດການແລ້ວ

ເຄື່ອງມືບໍ່ໄດ້ພຽງແຕ່ແປຂໍ້ຄວາມ — ມັນຮູ້ວ່າລະບົບການຂຽນແຕ່ລະອັນສະແດງແນວໃດ:

| ຕົວໜັງສື | ສິ່ງທີ່ປ່ຽນ | ເປັນຫຍັງ |
|---|---|---|

## ການເກັບຂໍ້ມູນ

ການແປແມ່ນຜະລິດຕະພັນຈາກການສ້າງ. ສ້າງໃນເວລາສ້າງ, ເກັບຜົນໄດ້ຮັບໄວ້, ຂ້າມເມື່ອແຫຼ່ງບໍ່ໄດ້ປ່ຽນແປງ.

### Vercel

Vercel ເກັບຜົນໄດ້ຮັບຈາກການສ້າງໂດຍອັດຕະໂນມັດ. ເພີ່ມ `postbuild` ແລະທ່ານເສັດແລ້ວ:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

ແຫຼ່ງບໍ່ໄດ້ປ່ຽນແປງ = cache hit = ສູນການເອີ້ນ LLM = ສູນຄ່າໃຊ້ຈ່າຍ.

## ຕົວເລືອກ

```
ການໃຊ້: translator-agent [ຕົວເລືອກ]

ຕົວເລືອກ:
  -s, --source <path>      ໄດເຣັກເຕີຣີຫຼືໄຟລ໌ແຫຼ່ງທີ່ຈະສະແກນ
  -l, --locales <locales>  ທ້ອງຖິ່ນເປົ້າໝາຍ, ແຍກດ້ວຍຈຸນລະພາກຫຼື "all" ສໍາລັບ 71 ພາສາ
  -o, --output <path>      ໄດເຣັກເຕີຣີຜົນໄດ້ຮັບ (ຄ່າເລີ່ມຕົ້ນ: "./translations")
  -p, --provider <name>    anthropic | openai (ຄ່າເລີ່ມຕົ້ນ: "anthropic")
  -m, --model <id>         ການແທນທີ່ໂມເດວ
  -c, --concurrency <n>    ການເອີ້ນ LLM ພ້ອມກັນສູງສຸດ (ຄ່າເລີ່ມຕົ້ນ: 10)
  --api-url <url>          URL ບໍລິການ hosted (ໃຊ້ອັດຕະໂນມັດເມື່ອບໍ່ໄດ້ຕັ້ງກະແຈ API)
```

| Extension | ຍຸດທະສາດ |
|---|---|
| `.json` | ແປຄ່າ, ຮັກສາກະແຈ |
| `.md` / `.mdx` | ແປຂໍ້ຄວາມ, ຮັກສາ syntax |
| `.html` / `.htm` | ແປຂໍ້ຄວາມ, ຮັກສາແທັກ, ແຊກ `lang`/`dir` |
| ອັນອື່ນໆທັງໝົດ | ກ໊ອບປີໃສ່ໄດເຣັກເຕີຣີແຕ່ລະທ້ອງຖິ່ນ |

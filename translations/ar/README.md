# translator-agent

مشكلة التوطين بقيمة 10,000 دولار، تم حلها في 90 ثانية.

تدفع الشركات للوكالات 0.10-0.25 دولار للكلمة الواحدة لتوطين مواقعها. موقع من 5,000 كلمة إلى 10 لغات يكلف 5,000-12,000 دولار ويستغرق 2-4 أسابيع. كلما غيرت عنواناً، يعود العداد للصفر.

هذه الأداة تقوم بذلك في أمر واحد، إلى 71 لغة، خلال مرحلة البناء:

```bash
bunx translator-agent -s ./dist -l all
```

بلا وكالة. بلا جداول بيانات. بلا حبس بائع. بلا تسجيل. مفاتيحك، برمجتك، لغاتك.

## لماذا هذا ناجح

الترجمة مسألة محلولة. التوطين ليس كذلك.

Google Translate يمكنها تحويل "الهامسترات تعمل على الأمر" إلى اليابانية. ما لا تستطيع فعله هو أن تدرك أن النكتة لا تنجح في اليابان، وتستبدلها بشيء ينجح — مثل الإشارة لفريق الهندسة الذي يسهر الليل كله، وهو مناسب ثقافياً ومضحك في السياق.

هذه الأداة لا تترجم. إنها **تعيد الإبداع** — نفس العملية التي تتقاضى عليها وكالات الإعلان 50,000 دولار عندما تعدل حملة عبر الأسواق. إلا أن النموذج اللغوي يعرف بالفعل كل ثقافة، كل مصطلح، كل اصطلاح تنسيق. يعرف أن:

- `49 دولار/شهر` تصبح `月額6,980円` في اليابان — وليس "49 دولار" برمز الين ملصق عليها
- السخرية تنجح بالإنجليزية وتموت باليابانية الرسمية
- "الغرق في الأوراق" تصبح "noyade administrative" بالفرنسية — تعبير فرنسي حقيقي، وليس ترجمة حرفية
- الألمان يحتفظون بنكتة الهامستر لأن Hamsterrad (عجلة الهامستر) مصطلح ألماني حقيقي
- البرازيليون يحتاجون الأسلوب غير الرسمي وإلا بدا وكأن روبوت كتبه

النموذج يصنف كل نص. تسميات واجهة المستخدم تحصل على ترجمة مباشرة. النسخ التسويقية تتم مواءمتها. الفكاهة تُعاد كتابتها كلياً للثقافة المستهدفة.

## ما يحدث عندما تشغلها

وجهها إلى مخرجات البناء. تستنسخ شجرة الملفات كاملة لكل محلي — تترجم الملفات النصية، تنسخ الأصول الثابتة، وتولد كل ما يحتاج للنشر:

```
your-site/                          translations/
  index.html                          middleware.ts        ← كشف المحلي
  about.html             →            _locales.css         ← طباعة لكل نص
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja"، معاد الإبداع
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP، ارتفاع خط 1.8
                                        css/style.css      ← منسوخ
                                        js/app.js          ← منسوخ
                                        images/logo.png    ← منسوخ
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic، خط 110%
                                        ...
                                      de/
                                        ...
```

كل ملف HTML يحصل على `lang` و`dir="rtl"` محقون. كل محلي يحصل على CSS مع مجموعة الخطوط الصحيحة، ارتفاع الخط، واتجاه النص. يتم توليد وسطاء Vercel يقرأ `Accept-Language` ويعيد الكتابة للمحلي المناسب.

انشر على Vercel. مستخدم في طوكيو يرى اليابانية مع Hiragino Sans بارتفاع خط 1.8. مستخدم في القاهرة يرى العربية RTL مع Noto Naskh بحجم 110%. مستخدم في بانكوك يرى التايلاندية مع `word-break: keep-all` لأن التايلاندية لا مسافات فيها. بلا إعداد.

## 90 ثانية، وليس 4 أسابيع

```bash
# ثلاث لغات، ملف JSON واحد
$ bunx translator-agent -s ./messages.json -l fr,de,ja
  [33%] fr/messages.json
  [67%] de/messages.json
  [100%] ja/messages.json
تم. 3 ملفات مكتوبة في 9.5 ثانية

# موقعك كاملاً، كل لغة على الأرض
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
تم. 142 ملف مترجم، 284 ملف ثابت منسوخ في 94 ثانية
```

### في خط إنتاجك

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

كل نشر يخرج بـ 71 لغة. الترجمات هي مصنوعات بناء — مخزنة مؤقتاً، تُعاد إنتاجها فقط عند تغيير المصدر.

## أحضر مفاتيحك أو لا تفعل

```bash
# لديك مفاتيح — يعمل محلياً، تدفع لمزود النموذج اللغوي مباشرة
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# ليس لديك مفاتيح — يعمل فقط
# يستخدم تلقائياً الخدمة المستضافة
# ادفع لكل ترجمة بـ USDC عبر x402 — بلا تسجيل، بلا حساب
bunx translator-agent -s ./dist -l all
```

نفس الأمر. إذا كانت مفاتيح API موجودة، يعمل محلياً مع مزودك. إذا لم تكن، يصل للـ API المستضاف ويدفع لكل طلب عبر [x402](https://x402.org) — بروتوكول دفع HTTP 402. عميلك يدفع USDC على Base، يحصل على ترجمات. بلا مصادقة، بلا علاقة بائع، بلا فواتير.

يدعم Anthropic و OpenAI. أحضر أي نموذج تريد:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## كل نظام كتابة، مُتعامل معه

الأداة لا تترجم النص فقط — تعرف كيف يُعرض كل نظام كتابة:

| النص | ما يتغير | لماذا |
|---|---|---|
| **العربية، العبرية، الفارسية، الأردية** | `dir="rtl"`، خطوط RTL، حجم 110% | العربية تحتاج خط أكبر لتكون واضحة؛ التخطيط كله ينعكس |
| **اليابانية، الصينية، الكورية** | مجموعات خطوط CJK، ارتفاع خط 1.8 | الأحرف مربعات عرض ثابت؛ تحتاج مسافة تنفس عمودية |
| **الهندية، البنغالية، التاميل، التيلوغو** | خطوط هندية، ارتفاع خط 1.8 | الخطوط العلوية (shirorekha) تحتاج مسافة عمودية إضافية |
| **التايلاندية** | `word-break: keep-all` | لا مسافات بين الكلمات — المتصفح يحتاج قواعد كسر خط صريحة |
| **البورمية** | ارتفاع خط 2.2 | أطول حروف في أي نص رئيسي |
| **الخمير** | ارتفاع خط 2.0 | مجموعات الساكن التحتي تتراكم عمودياً |

CSS المولد لكل محلي:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## التخزين المؤقت

الترجمات هي مصنوعات بناء. تولد وقت البناء، تخزن المخرجات مؤقتاً، تتخطى عندما لا يتغير المصدر.

### Vercel

Vercel يخزن مخرجات البناء تلقائياً. أضف `postbuild` وانتهيت:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

المصدر لم يتغير = إصابة في التخزين المؤقت = صفر استدعاءات نموذج لغوي = صفر تكلفة.

## الخيارات

```
الاستخدام: translator-agent [خيارات]

الخيارات:
  -s, --source <مسار>      المجلد أو الملف المصدر للمسح
  -l, --locales <محليات>  المحليات المستهدفة، مفصولة بفاصلة أو "all" لـ 71 لغة
  -o, --output <مسار>      مجلد المخرجات (افتراضي: "./translations")
  -p, --provider <اسم>     anthropic | openai (افتراضي: "anthropic")
  -m, --model <معرف>       تجاوز النموذج
  -c, --concurrency <ن>    أقصى استدعاءات نموذج لغوي متوازية (افتراضي: 10)
  --api-url <رابط>         رابط الخدمة المستضافة (يُستخدم تلقائياً عند عدم وجود مفاتيح API)
```

| الامتداد | الاستراتيجية |
|---|---|
| `.json` | ترجم القيم، احتفظ بالمفاتيح |
| `.md` / `.mdx` | ترجم النص، احتفظ بالصيغة |
| `.html` / `.htm` | ترجم النص، احتفظ بالتاغات، احقن `lang`/`dir` |
| كل شيء آخر | انسخ في كل مجلد محلي |

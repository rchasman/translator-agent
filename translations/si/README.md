# translator-agent

ඩොලර් 10,000 දේශීයකරණ ගැටලුව, තත්පර 90 කින් විසඳයි.

සමාගම් ඔවුන්ගේ වෙබ් අඩවි දේශීයකරණය සඳහා ඒජන්සිවලට වචනයකට ඩොලර් 0.10–0.25 ගෙවයි. වචන 5,000 ක වෙබ් අඩවියක් භාෂා 10 කට වෙනස් කිරීම සඳහා ඩොලර් 5,000–12,000 ක් වැයවන අතර සති 2–4 ක් ගතවේ. ඔබ සිරස්තලයක් වෙනස් කරන සෑම අවස්ථාවකම මීටරය නැවත ආරම්භ වේ.

මෙම මෙවලම එය එක් විධානයකින්, භාෂා 71 කට, ඔබේ ගොඩනැගීමේ පියවරේදීම සිදු කරයි:

```bash
bunx translator-agent -s ./dist -l all
```

ඒජන්සියක් නැත. ස්ප්‍රෙඩ්ශීට් නැත. විකුණුම්කරු කොටු වීමක් නැත. ලියාපදිංචි වීමක් නැත. ඔබේ යතුරු, ඔබේ ගොඩනැගීම, ඔබේ භාෂා.

## මෙය ක්‍රියාත්මක වන්නේ ඇයි

පරිවර්තනය විසඳුණු ගැටලුවකි. දේශීයකරණය එසේ නොවේ.

Google Translate හට "අපේ හැම්ස්ටර්ලා ඒක මත වැඩ කරනවා" ජපන් බසට පරිවර්තනය කළ හැකිය. එයට කළ නොහැකි වන්නේ විහිළුව ජපානයේ ලක් නොවන බව හඳුනා ගෙන, එය වේගයට කරන දෙයකින් — ඉංජිනේරු කණ්ඩායම සමස්ත රාත්‍රියක් ගත කරන ගැන සඳහන් කිරීම වැනි, සංස්කෘතිකව සුදුසු සහ සන්දර්භයේ විනෝදජනක වන දෙයකින් — ප්‍රතිස්ථාපනය කිරීමයි.

මෙම මෙවලම පරිවර්තනය නොකරයි. එය **අන්තර්ගත නිර්මාණය** කරයි — වෙළඳ ප්‍රචාරණ ඒජන්සි මාර්කට් හරහා ව්‍යාපාරයක් අනුවර්තනය කරන විට ඩොලර් 50,000 කට ගබඩා කරන ක්‍රියාවලිය. LLM දැනටමත් සෑම සංස්කෘතියක්ම, සෑම වාග් රූඪියක්ම, සෑම ෆෝමැටිං සම්මුතියක්ම දන්නා හේතු නිසා. එය අවබෝධ කරගනී:

- `$49/month` ජපානයේ `月額6,980円` බවට පරිණත වන්නේ — yen සංකේතයක් ගසන "$49" නොව
- උපහාසය ඉංග්‍රීසියෙන් හොඳ ඵලයක් දෙන්නේ අතර ධාතුමය ජපන් භාෂාවෙන් මිය යනවා
- "කඩදාසි වලංගු වීම" ප්‍රංශයේ "noyade administrative" බවට පරිණත වේ — සත්‍ය ප්‍රංශ ප්‍රකාශනයක්, වචනයෙන් වචනයට පරිවර්තනයක් නොව
- ජර්මන්කරුවන් හැම්ස්ටර් විහිළුව තබා ගන්නේ Hamsterrad (hamster wheel) සත්‍ය ජර්මන් වාග් රූඪියක් නිසා
- බ්‍රසීලියානුවන්ට අස්ථිර ලේඛනයක් අවශ්‍ය නැත්නම් එය රොබෝවෙකු ලිව්වාක් මෙන් ශබ්ද වේ

ආකෘතිය සෑම ස්ට්‍රිංගයක්ම වර්ගීකරණය කරයි. UI ලේබල් සෘජු පරිවර්තනය ලබයි. අලෙවිකරණ පිටපතක් අනුවර්තනය වේ. හාස්‍යය ඉලක්ක සංස්කෘතිය සඳහා සම්පූර්ණයෙන්ම නැවත නිර්මාණය වේ.

## ඔබ එය ධාවනය කරන විට සිදු වන්නේ කුමක්ද

ඔබේ ගොඩනැගීමේ ප්‍රතිදානය වෙත එය යොමු කරන්න. එය සෑම ලොකේලයකටම සම්පූර්ණ ගොනු ගසක්ම ක්ලෝන් කරයි — පෙළ ගොනු පරිවර්තනය කරමින්, ස්ථිර වත්කම් පිටපත් කරමින්, හා යෙදීම සඳහා අවශ්‍ය සියල්ල ජනනය කරමින්:

```
your-site/                          translations/
  index.html                          middleware.ts        ← ලොකේල හඳුනාගැනීම
  about.html             →            _locales.css         ← පිටපතට අනුව අකුරුකේත
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", අන්තර්ගත නිර්මාණය
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 පදකම් උස
                                        css/style.css      ← පිටපත්
                                        js/app.js          ← පිටපත්
                                        images/logo.png    ← පිටපත්
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% අකුරු
                                        ...
                                      de/
                                        ...
```

සෑම HTML ගොනුවකටම `lang` සහ `dir="rtl"` එන්ජෙක්ට් වේ. සෑම ලොකේලයකටම නියම අකුරු ගොඩ, පදකම් උස, සහ පෙළ දිශාව සමඟ CSS ලැබේ. `Accept-Language` කියවන සහ නියම ලොකේලයට ප්‍රතිලේඛනය කරන Vercel මිඩ්ල්වෙයාර් ජනනය වේ.

Vercel වෙත යවන්න. ටෝකියෝ හි පරිශීලකයෙකු 1.8 පදකම් උසින් Hiragino Sans සමග ජපන් දකී. කයිරෝ හි පරිශීලකයෙකු 110% ප්‍රමාණයෙන් Noto Naskh සමග RTL අරාබි දකී. බැංකොක් හි පරිශීලකයෙකු `word-break: keep-all` සමග තායි දකින්නේ තායි භාෂාවේ ඉඩකඩ නැති නිසාය. වින්‍යාසයක් නැත.

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

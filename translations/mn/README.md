# translator-agent

$10,000-ийн орчуулгын асуудал 90 секундэд шийдэгдлээ.

Компаниуд агентлагуудад өөрсдийн сайтуудыг орчуулахын тулд нэг үгэнд $0.10–0.25 төлдөг. 5,000 үгтэй сайтыг 10 хэл рүү орчуулахад $5,000–12,000 зарцуулж, 2–4 долоо хоног хугацаа шаарддаг. Гарчиг солих болгонд тоолуур дахин эхэлдэг.

Энэ хэрэгсэл үүнийг нэг тушаалаар, 71 хэл рүү, таны бүтээн байгуулалтын шатанд хийдэг:

```bash
bunx translator-agent -s ./dist -l all
```

Агентлаг байхгүй. Хүснэгт байхгүй. Борлуулагчтай холбогдохгүй. Бүртгэл байхгүй. Таны түлхүүр, таны бүтээн байгуулалт, таны хэлнүүд.

> **Та нотолгоог уншиж байна.** Энэ README-г `bunx translator-agent -s README.md -l all` ажиллуулж орчуулсан. [Бразил португал хувилбар](./translations/pt-BR/README.md) нь бразилчууд бичсэн мэт сонсогддог, яагаад гэвэл энэ нь гол зорилго юм.

## Яагаад энэ нь ажилладаг вэ

Орчуулга бол шийдэгдсэн асуудал. Орон нутагжуулалт тийм биш.

Google Translate нь "Манай хомягууд түүн дээр ажиллаж байна"-г япон хэл рүү орчуулж чадна. Гэхдээ энэ хошигнол Японд дуусгахгүй болохыг мэдэж, түүний оронд бусад зүйлээр солих — инженерийн багийнхан шөнөжин ажиллаж байгааг дурдах зэрэг, энэ нь соёлын хувьд тохиромжтой бөгөөд нөхцөлд тохирсон инээдтэй байдаг.

Энэ хэрэгсэл орчуулдаггүй. Энэ нь **дахин бүтээдэг** — зар сурталчилгааны агентлагууд кампанит ажлыг зах зээлүүдэд дасан зохицуулахдаа $50,000 төлдөг ижил үйл явц. Гэхдээ LLM нь аль хэдийн бүх соёл, бүх хэлц үг, бүх форматлах конвенцийг мэддэг. Энэ нь мэддэг:

- `$49/month` нь Японд `月額6,980円` болдог — "$49" дээр иен тэмдэг нааж өгсөн биш
- Ёжлол англи хэлэнд амжилттай боловч албан ёсны япон хэлэнд үхдэг
- "Drowning in paperwork" нь франц хэлэнд "noyade administrative" болдог — үгийг үг рүү орчуулсан биш, жинхэнэ франц хэллэг
- Германчууд хомягын хошигнолыг хадгалдаг учир нь Hamsterrad (хомягын дугуй) бол жинхэнэ герман хэлц үг
- Бразилчууд энгийн аягтай байх шаардлагатай, эс бөгөөс робот бичсэн мэт сонсогддог

Загвар нь мөр тус бүрийг ангилдаг. UI шошго нь шууд орчуулгатай. Маркетингийн хуулбар дасан зохицдог. Хөгжилтэй зүйлс зорилтот соёлд бүрэн дахин бүтээгддэг.

## Та үүнийг ажиллуулахад юу тохиолддог вэ

Таны бүтээн байгуулалтын гаралт руу заана уу. Энэ нь бүх файлын модыг орон нутгийн хувьд хуулж авдаг — текст файлуудыг орчуулж, статик хөрөнгүүдийг хуулж, байршуулалтанд шаардлагатай бүх зүйлийг үүсгэдэг:

```
your-site/                          translations/
  index.html                          middleware.ts        ← орон нутгийг илрүүлэх
  about.html             →            _locales.css         ← скриптийн хичээл бичих
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", дахин бүтээсэн
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 мөрийн өндөр
                                        css/style.css      ← хуулсан
                                        js/app.js          ← хуулсан
                                        images/logo.png    ← хуулсан
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% үсэг
                                        ...
                                      de/
                                        ...
```

HTML файл бүрт `lang` болон `dir="rtl"` оруулагддаг. Орон нутаг бүр зөв үсгийн стек, мөрийн өндөр, текстийн чиглэлтэй CSS авдаг. Vercel дундын програм хангамж үүсгэгдэж, `Accept-Language`-г уншиж, зөв орон нутаг руу дахин бичдэг.

Vercel дээр байршуулах. Токио дахь хэрэглэгч Hiragino Sans-ээр 1.8 мөрийн өндөртэй япон хэлийг харна. Каир дахь хэрэглэгч Noto Naskh-ээр 110% хэмжээтэй RTL араб хэлийг харна. Бангкок дахь хэрэглэгч Тай хэлд зай байхгүй учир `word-break: keep-all`-тай Тай хэлийг харна. Тохиргоо байхгүй.

## 90 секунд, 4 долоо хоног биш

```bash
# Гурван хэл, нэг JSON файл
$ bunx translator-agent -s ./messages.json -l fr,de,ja

  [67%] de/messages.json
  [100%] ja/messages.json
Дууслаа. 3 файл 9.5 секундэд бичигдсэн

# Таны бүх сайт, дэлхий дээрх бүх хэл
$ bunx translator-agent -s ./dist -l all
  [1%] zh-CN/index.html
  ...
  [100%] mn/about.html
Дууслаа. 142 файл орчуулагдсан, 284 статик файл 94 секундэд хуулагдсан
```

## Өөрийн түлхүүрээ авчирах эсвэл битгий

```bash
# Танд түлхүүр байна — орон нутагт ажиллаж, LLM үйлчилгээ үзүүлэгчдээ шууд төлдөг
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# Танд түлхүүр байхгүй — зүгээр л ажилладаг
# Хостлогдсон үйлчилгээг автоматаар ашиглана
# x402-ээр USDC-ээр орчуулгын төлбөр төлөх — бүртгэл, данс байхгүй
bunx translator-agent -s ./dist -l all
```

Ижил тушаал. API түлхүүрүүд байвал таны үйлчилгээ үзүүлэгчтэй орон нутагт ажилладаг. Үгүй бол хостлогдсон API руу хандаж, [x402](https://x402.org)-ээр хүсэлт тус бүрийн төлбөр төлдөг — HTTP 402 төлбөрийн протокол. Таны үйлчлүүлэгч Base дээр USDC төлж, орчуулга буцааж авна. Баталгаажуулалт, борлуулагчийн харилцаа, нэхэмжлэх байхгүй.

Anthropic болон OpenAI-г дэмждэг. Хүссэн загваргаа авчирна уу:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## Бүх скрипт систем, зохицуулсан

Энэ хэрэгсэл зөвхөн текстийг орчуулдаггүй — бичих систем бүрийг хэрхэн дүрслэхийг мэддэг:

| Скрипт | Юу өөрчлөгддөг | Яагаад |
|---|---|---|
| **Араб, еврей, фарс, урду** | `dir="rtl"`, RTL фонтууд, 110% хэмжээ | Араб хэл уншихын тулд том үсэг хэрэгтэй; бүх зохион байгуулалт тусгалдана |
| **Япон, хятад, солонгос** | CJK фонтын стекүүд, 1.8 мөрийн өндөр | Тэмдэгтүүд нь тогтсон өргөнийхөүүд; босоо амьсгалах зай хэрэгтэй |
| **Хинди, бенгал, тамил, телугу** | Энэтхэгийн фонтууд, 1.8 мөрийн өндөр | Толгойны зурас (широрекха) нэмэлт босоо зай хэрэгтэй |
| **Тай** | `word-break: keep-all` | Үгсийн хооронд зай байхгүй — хөтчид тодорхой мөр таслах дүрэм хэрэгтэй |
| **Бирм** | 2.2 мөрийн өндөр | Гол скриптүүдийн хамгийн өндөр глифүүд |
| **Кхмер** | 2.0 мөрийн өндөр | Дэд гийгүүлэгчийн багц нь босоо овоолдог |

Орон нутаг тус бүрийн үүсгэсэн CSS:

```css
/* ar/_locale.css */
body {
  font-family: "Noto Naskh Arabic", "Geeza Pro", Tahoma, sans-serif;
  line-height: 1.9;
  font-size: 110%;
}
[dir="rtl"] { text-align: right; }
```

## Кэшлэх

Орчуулга нь бүтээн байгуулалтын гаргалга. Бүтээн байгуулалтын цагт үүсгэж, гаргалгыг кэшлэж, эх өөрчлөгдөөгүй бол алгасна.

### Vercel

Vercel нь бүтээн байгуулалтын гаралтыг автоматаар кэшлдэг. `postbuild` нэмж, дууслаа:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Эх өөрчлөгдөөгүй = кэш оноосон = тэг LLM дуудлага = тэг өртөг.

## Сонголтууд

```
Хэрэглээ: translator-agent [сонголтууд]

Сонголтууд:
  -s, --source <зам>      сканнердах эх лавлах эсвэл файл
  -l, --locales <орон нутгууд>  зорилтот орон нутгууд, таслалаар тусгаарлагдсан эсвэл 71 хэлийн хувьд "all"
  -o, --output <зам>      гаргалгын лавлах (анхдагч: "./translations")
  -p, --provider <нэр>    anthropic | openai (анхдагч: "anthropic")
  -m, --model <id>        загварын дарангуйлал
  -c, --concurrency <n>    хамгийн их зэрэг LLM дуудлага (анхдагч: 10)
  --api-url <url>          хостлогдсон үйлчилгээний URL (API түлхүүрүүд тохируулаагүй үед автоматаар ашиглагдана)
```

| Өргөтгөл | Стратеги |
|---|---|
| `.json` | Утгуудыг орчуулах, түлхүүрүүдийг хадгалах |
| `.md` / `.mdx` | Текстийг орчуулах, синтаксийг хадгалах |
| `.html` / `.htm` | Текстийг орчуулах, шошгыг хадгалах, `lang`/`dir` оруулах |
| Бусад бүгд | Орон нутаг бүрийн лавлахад хуулах |

### Бүх 71 орон нутаг

`-l all` нь интернет хэрэглэгчдийн ~95%-ийг хамардаг: zh-CN, zh-TW, ja, ko, vi, th, id, ms, fil, my, hi, bn, ta, te, mr, gu, kn, ml, pa, ur, fa, tr, he, ar, kk, uz, fr, de, es, pt, pt-BR, it, nl, ca, gl, sv, da, no, fi, is, pl, cs, sk, hu, ro, bg, hr, sr, sl, uk, ru, lt, lv, et, el, ga, sw, am, ha, yo, zu, af, km, lo, ne, si, ka, az, mn

₹8,26,000 പ്രൊലോക്കലൈസേഷൻ പ്രശ്നം, 90 സെക്കൻഡിൽ പരിഹരിച്ചു.

കമ്പനികൾ തങ്ങളുടെ സൈറ്റുകൾ പ്രാദേശികവത്കരിക്കാൻ ഏജൻസികൾക്ക് വാക്കിന് ₹8.25–20.66 നൽകുന്നു. 5,000 വാക്കുകളുള്ള ഒരു സൈറ്റ് 10 ഭാഷകളിലേക്ക് മാറ്റുന്നതിന് ₹4,13,000–9,91,200 ചിലവാകുന്നു, 2–4 ആഴ്ച എടുക്കുന്നു. നിങ്ങൾ ഒരു തലക്കെട്ട് മാറ്റുമ്പോഴൊക്കെ, മീറ്റർ വീണ്ടും തുടങ്ങുന്നു.

ഈ ടൂൾ ഒരു കമാൻഡിൽ, 71 ഭാഷകളിലേക്ക്, നിങ്ങളുടെ ബിൽഡ് സ്റ്റെപ്പിൽ തന്നെ ചെയ്യുന്നു:

```bash
bunx translator-agent -s ./dist -l all
```

## ഇത് എന്തുകൊണ്ട് വർക്ക് ചെയ്യുന്നു

വിവർത്തനം പരിഹരിക്കപ്പെട്ട പ്രശ്നമാണ്. പ്രാദേശികവത്കരണം അല്ല.

Google Translate-ന് "Our hamsters are working on it" ജപ്പാനീസിലേക്ക് മാറ്റാം. അതിന് ചെയ്യാൻ കഴിയാത്തത് ജപ്പാനിൽ ആ തമാശ പിടിക്കില്ലെന്ന് തിരിച്ചറിയുകയും പകരം വല്ലതും വയ്ക്കുകയും ചെയ്യുക — എഞ്ചിനീയറിംഗ് ടീം രാത്രി മുഴുവൻ ജോലി ചെയ്യുന്നതിനെ പറ്റി പറയുക, അത് സാംസ്കാരികമായി ഉചിതവും സന്ദർഭത്തിൽ രസകരവുമാണ്.

ഈ ടൂൾ വിവർത്തനം ചെയ്യുന്നില്ല. അത് **ട്രാൻസ്ക്രിയേറ്റ്** ചെയ്യുന്നു — പ്രചാരണം വിവിധ വിപണികളിലാക്കി മാറ്റുമ്പോൾ പരസ്യ ഏജൻസികൾ $50,000 ചാർജ് ചെയ്യുന്ന അതേ പ്രക്രിയ. എന്നാൽ LLM-ന് ഓരോ സംസ്കാരം, ഓരോ പഴഞ്ചൊല്ല്, ഓരോ ഫോർമാറ്റിംഗ് നിയമാവലി എന്നിവ അറിയാം. അതിനറിയാം:

- `$49/month` ജപ്പാനിൽ `月額6,980円` ആയിത്തീരുന്നു — യെൻ ചിഹ്നം പുരട്ടിയ "$49" അല്ല
- സാർകാസം ഇംഗ്ലീഷിൽ കൊള്ളാം, ഫോർമൽ ജപ്പാനീസിൽ ചത്തുപോകും
- "Drowning in paperwork" ഫ്രഞ്ചിൽ "noyade administrative" ആകുന്നു — ഒരു യഥാർത്ഥ ഫ്രഞ്ച് പ്രയോഗം, വാക്കിന് വാക്ക് വിവർത്തനം അല്ല
- ജർമ്മൻകാർ ഹാംസ്റ്റർ തമാശ കൊണ്ടുപോകുന്നു കാരണം Hamsterrad (hamster wheel) ഒരു യഥാർത്ഥ ജർമ്മൻ പഴഞ്ചൊല്ലാണ്
- ബ്രസീലിയൻകാർക്ക് casual register വേണം അല്ലെങ്കിൽ ഒരു റോബോട്ട് എഴുതിയത് പോലെ തോന്നും

മോഡൽ ഓരോ സ്ട്രിംഗും ക്ലാസിഫൈ ചെയ്യുന്നു. UI ലേബലുകൾക്ക് നേരിട്ടുള്ള വിവർത്തനം. മാർക്കറ്റിംഗ് കോപ്പി അഡാപ്റ്റ് ചെയ്യപ്പെടുന്നു. ഹ്യൂമർ ടാർഗെറ്റ് സംസ്കാരത്തിനായി പൂർണമായും പുനഃസൃഷ്ടിക്കപ്പെടുന്നു.

## നിങ്ങൾ ഇത് റൺ ചെയ്യുമ്പോൾ എന്ത് സംഭവിക്കുന്നു

നിങ്ങളുടെ ബിൽഡ് ഔട്ട്പുട്ടിലേക്ക് പോയിന്റ് ചെയ്യുക. അത് ഓരോ ലൊക്കേലിലും മുഴുവൻ ഫൈൽ ട്രീ ക്ലോൺ ചെയ്യുന്നു — ടെക്സ്റ്റ് ഫൈലുകൾ വിവർത്തനം ചെയ്ത്, സ്റ്റാറ്റിക് അസ്സറ്റുകൾ കോപ്പി ചെയ്ത്, ഡിപ്ലോയ്മെന്റിന് ആവശ്യമായ എല്ലാം ജനറേറ്റ് ചെയ്ത്:

```
your-site/                          translations/
  index.html                          middleware.ts        ← locale detection
  about.html             →            _locales.css         ← typography per script
  css/style.css                       ja/
  js/app.js                             index.html         ← lang="ja", transcreated
  images/logo.png                       about.html
                                        _locale.css        ← Noto Sans JP, 1.8 line-height
                                        css/style.css      ← copied
                                        js/app.js          ← copied
                                        images/logo.png    ← copied
                                      ar/
                                        index.html         ← lang="ar" dir="rtl"
                                        _locale.css        ← Noto Naskh Arabic, 110% font
                                        ...
                                      de/
                                        ...
```

ഓരോ HTML ഫൈലിലും `lang`, `dir="rtl"` എന്നിവ ഇഞ്ചക്റ്റ് ചെയ്യപ്പെടുന്നു. ഓരോ ലൊക്കേലിനും ശരിയായ ഫോണ്ട് സ്റ്റാക്ക്, ലൈൻ-ഹൈറ്റ്, ടെക്സ്റ്റ് ദിശ എന്നിവയുള്ള CSS ലഭിക്കുന്നു. `Accept-Language` വായിച്ച് ശരിയായ ലൊക്കേലിലേക്ക് റീറൈറ്റ് ചെയ്യുന്ന Vercel മിഡിൽവെയർ ജനറേറ്റ് ചെയ്യപ്പെടുന്നു.

Vercel-ൽ ഡിപ്ലോയ് ചെയ്യുക. ടോക്യോയിലെ ഒരു യൂസർ 1.8 ലൈൻ-ഹൈറ്റിൽ Hiragino Sans-ൽ ജപ്പാനീസ് കാണുന്നു. കെയ്റോയിലെ ഒരു യൂസർ 110% സൈസിൽ Noto Naskh-ൽ RTL അറബിക് കാണുന്നു. ബാംഗ്കോക്കിലെ ഒരു യൂസർ `word-break: keep-all`-ൽ തായ് കാണുന്നു കാരണം തായിയിൽ സ്പേസുകൾ ഇല്ല. കോൺഫിഗ് ഇല്ല.

## സ്വന്തം കീകൾ കൊണ്ടുവരൂ അല്ലെങ്കിൽ വേണ്ട

```bash
# നിങ്ങളുടെ പക്കൽ കീകൾ ഉണ്ട് — ലോക്കലായി റൺ ചെയ്യുന്നു, നിങ്ങൾ നിങ്ങളുടെ LLM പ്രൊവൈഡറിന് നേരിട്ട് പണം നൽകുന്നു
export ANTHROPIC_API_KEY=sk-...
bunx translator-agent -s ./dist -l all

# നിങ്ങളുടെ പക്കൽ കീകൾ ഇല്ല — സിമ്പിൾ ആയി വർക്ക് ചെയ്യുന്നു
# ഹോസ്റ്റ് ചെയ്ത സെർവിസ് ഓട്ടോമാറ്റിക്കായി ഉപയോഗിക്കുന്നു
# x402 വഴി USDC-യിൽ ഓരോ വിവർത്തനത്തിനും പണം നൽകുന്നു — സൈൻഅപ്പ് ഇല്ല, അക്കൗണ്ട് ഇല്ല
bunx translator-agent -s ./dist -l all
```

അതേ കമാൻഡ്. API കീകൾ ഉണ്ടെങ്കിൽ, അത് നിങ്ങളുടെ പ്രൊവൈഡറിൽ ലോക്കലായി റൺ ചെയ്യുന്നു. ഇല്ലെങ്കിൽ, അത് ഹോസ്റ്റ് ചെയ്ത API ഹിറ്റ് ചെയ്ത് [x402](https://x402.org) വഴി ഓരോ അഭ്യർത്ഥനക്കും പണം നൽകുന്നു — HTTP 402 പേയ്മെന്റ് പ്രോട്ടോക്കോൾ. നിങ്ങളുടെ ക്ലയന്റ് Base-ൽ USDC നൽകുന്നു, വിവർത്തനങ്ങൾ തിരിച്ചുകിട്ടുന്നു. ഓത്തറൈസേഷൻ ഇല്ല, വെണ്ടർ ബന്ധം ഇല്ല, ഇൻവോയ്സുകൾ ഇല്ല.

Anthropic, OpenAI എന്നിവയെ സപ്പോർട്ട് ചെയ്യുന്നു. നിങ്ങൾക്ക് വേണ്ട ഏത് മോഡലും കൊണ്ടുവരൂ:

```bash
bunx translator-agent -s ./dist -l all -p openai -m gpt-4o
```

## എല്ലാ സ്ക്രിപ്റ്റ് സിസ്റ്റവും, ഹാൻഡിൽ ചെയ്യപ്പെട്ടിരിക്കുന്നു

ഈ ടൂൾ വെറും ടെക്സ്റ്റ് വിവർത്തനം ചെയ്യുന്നില്ല — ഓരോ എഴുത്ത് സമ്പ്രദായവും എങ്ങനെ റെൻഡർ ചെയ്യുന്നുവെന്ന് അറിയാം:

| സ്ക്രിപ്റ്റ് | എന്ത് മാറുന്നു | എന്തുകൊണ്ട് |
|---|---|---|
| **അറബിക്, ഹീബ്രു, ഫാർസി, ഉറുദു** | `dir="rtl"`, RTL ഫോണ്ടുകൾ, 110% സൈസ് | അറബിക്കിന് വായിക്കാൻ കൂടുതൽ ടൈപ്പ് സൈസ് വേണം; മുഴുവൻ ലേഔട്ടും മിറർ ചെയ്യുന്നു |
| **ജപ്പാനീസ്, ചൈനീസ്, കൊറിയൻ** | CJK ഫോണ്ട് സ്റ്റാക്കുകൾ, 1.8 ലൈൻ-ഹൈറ്റ് | അക്ഷരങ്ങൾ നിശ്ചിത വീതിയുള്ള ചതുരങ്ങളാണ്; വെർട്ടിക്കൽ ബ്രീതിംഗ് റൂം വേണം |

## കാഷിംഗ്

വിവർത്തനങ്ങൾ ബിൽഡ് ആർട്ടിഫാക്റ്റുകളാണ്. ബിൽഡ് ടൈമിൽ ജനറേറ്റ് ചെയ്യുക, ഔട്ട്പുട്ട് കാഷ് ചെയ്യുക, സോഴ്സ് മാറാത്തപ്പോൾ സ്കിപ്പ് ചെയ്യുക.

### Vercel

Vercel ബിൽഡ് ഔട്ട്പുട്ട് ഓട്ടോമാറ്റിക്കായി കാഷ് ചെയ്യുന്നു. `postbuild` ചേർക്കുക, നിങ്ങൾ പൂർത്തിയാക്കി:

```json
{
  "scripts": {
    "build": "next build",
    "postbuild": "bunx translator-agent -s ./public -l all -o ./public"
  }
}
```

### CI

```yaml
- uses: actions/cache@v4
  id: translations
  with:
    path: translations/
    key: translations-${{ hashFiles('src/messages/**', 'public/**/*.html') }}

- if: steps.translations.outputs.cache-hit != 'true'
  run: bunx translator-agent -s ./src/messages -l all
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

സോഴ്സ് മാറിയിട്ടില്ല = കാഷ് ഹിറ്റ് = പൂജ്യം LLM കോളുകൾ = പൂജ്യം ചിലവ്.

## ഓപ്ഷനുകൾ

```
Usage: translator-agent [options]

Options:
  -s, --source <path>      source directory or file to scan
  -l, --locales <locales>  target locales, comma-separated or "all" for 71 languages
  -o, --output <path>      output directory (default: "./translations")
  -p, --provider <name>    anthropic | openai (default: "anthropic")
  -m, --model <id>         model override
  -c, --concurrency <n>    max parallel LLM calls (default: 10)
  --api-url <url>          hosted service URL (auto-used when no API keys set)
```

| എക്സ്റ്റൻഷൻ | തന്ത്രം |
|---|---|
| `.json` | വാല്യൂകൾ വിവർത്തനം ചെയ്യുക, കീകൾ സംരക്ഷിക്കുക |
| `.md` / `.mdx` | ടെക്സ്റ്റ് വിവർത്തനം ചെയ്യുക, സിന്റാക്സ് സംരക്ഷിക്കുക |
| `.html` / `.htm` | ടെക്സ്റ്റ് വിവർത്തനം ചെയ്യുക, ടാഗുകൾ സംരക്ഷിക്കുക, `lang`/`dir` ഇഞ്ചക്റ്റ് ചെയ്യുക |
| എല്ലാ മറ്റുള്ളവയും | ഓരോ ലൊക്കേൽ ഡയറക്ടറിയിലേക്കും കോപ്പി ചെയ്യുക |
